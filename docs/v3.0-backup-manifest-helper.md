# v3.0 Backup Manifest Pure Helper

## Purpose

The backup manifest helper turns the v3.0 backup/rollback design into a future manifest shape that can be used by a later dry-run-only or write-capable import flow.

This helper is pure. It does not create backups, copy the DB, read or write the DB, run restore rehearsal, implement an import command, connect product import routes, or add API/UI/runtime behavior.

## Non-Goals

- Not actual backup creation.
- Not DB copy.
- Not DB read.
- Not DB write.
- Not restore rehearsal.
- Not import command implementation.
- Not product import route connection.
- Not API/UI integration.
- Not runtime wiring.

## Manifest Fields

The manifest shape includes:

- `version`
- `backupId`
- `timestampIso`
- `branchName`
- `commitHash`
- `dbFileName`
- `dbSha256Before`
- `importPreviewSummary`
- `validationSummary`
- `classificationSummary`
- `safetyChecks`
- `rollbackInstructions`
- `writeAllowed`
- `restoreRehearsalRequired`
- `warnings`

`timestampIso` and `backupId` are provided by the caller. The helper does not call `Date.now`, generate random ids, or inspect the filesystem.

## Safety Checks

`safetyChecks` records whether the future import preconditions passed:

- `gitStatusClean`
- `testPassed`
- `buildPassed`
- `smokePassed`
- `privacyScanPassed`
- `rawZipAbsent`
- `generatedArtifactJsonAbsent`
- `hrcDryRunReportsAbsent`
- `productImportRouteDisabled`
- `dbReadWriteNotPerformed`

If any check is false, the helper adds a warning and summary output reports the failed checks.

## Write Policy

`writeAllowed` is always `false`.

The manifest helper is a preview and policy artifact only. It cannot authorize a DB write and does not perform backup operations.

## Restore Rehearsal Policy

`restoreRehearsalRequired` is always `true`.

Any future write-capable import flow must have a documented restore rehearsal plan before DB write is considered.

## Local Path Redaction Policy

The helper sanitizes string fields and nested summary values so public-facing manifest output does not retain raw local path or account tokens.

Examples of redacted input include:

- Windows user paths.
- user tokens.
- sample-external-hrc-folder path fragments.
- raw HRC folder labels.
- email addresses.

`dbFileName` is reduced to a file name when a path-like input is supplied.
