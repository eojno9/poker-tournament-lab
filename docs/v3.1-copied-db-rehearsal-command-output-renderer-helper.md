# v3.1 Copied DB Rehearsal Command Output Renderer Helper

## Purpose

The copied DB rehearsal command output renderer helper turns a Step 6 copied DB rehearsal dry-run orchestrator result into safe CLI/console-oriented text. It returns strings only and does not print them.

## Non-goals

- No actual CLI entrypoint.
- No `package.json` script.
- No actual DB copy.
- No actual DB write.
- No production DB write.
- No DB schema migration.
- No report JSON file generation.
- No API/UI import flow.
- No product import route connection.
- No raw HRC zip access, copy, or extraction.
- No filesystem access.
- No DB access.
- No `process.env`, `Date.now`, random value, or console output side effect.

## Relationship to the Orchestrator

The renderer accepts the Step 6 orchestrator result shape. It does not rebuild approval, guard, plan, or report state. The Step 6 helper remains responsible for composing:

1. approval contract
2. rehearsal guard
3. rehearsal plan builder
4. rehearsal report shape

The renderer only formats the resulting status, decisions, counts, SHA summary, safety summary, execution summary, blocked reasons, warnings, and required next checks.

## Safe Console Output Policy

Rendered output includes:

- rehearsal id
- status
- exit code
- dry-run and copied DB rehearsal decisions
- `Can Production DB Write: false`
- `Can Write Report File: false`
- redacted target summary
- count summary
- SHA summary
- safety summary
- execution summary
- blocked reasons
- warnings
- required next checks

The helper returns:

- `lines`
- `text`
- `lineCount`
- `hasWarnings`
- `forbiddenExposureDetected`
- renderer warnings

The helper does not write files and does not call console output APIs.

## Forbidden Output Policy

The renderer redacts forbidden local/private output tokens before returning text. Forbidden tokens include local absolute paths, user/private token placeholders, email addresses, raw HRC wording, and raw external source folder placeholders.

If forbidden exposure is detected in raw render input, the returned text is redacted and `forbiddenExposureDetected` is true. The assertion helper can be used to fail a future command if the renderer had to redact unsafe text.

## Safety Display Policy

The renderer always displays production and report file write decisions as false:

- `Can Production DB Write: false`
- `Can Write Report File: false`

If the orchestrator report records `actualDbWritePerformed: true`, the renderer displays that evidence as blocked output. If the report records `reportJsonWritten: true`, the renderer also displays that evidence as blocked output.
