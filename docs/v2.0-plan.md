## UX Reference Direction

v2.0은 GTO Wizard, HRC Pro, ICMIZER, solver류 도구의 UX에서 다음 방향만 참고합니다.

- Spot을 먼저 고르고, matrix와 hand detail을 함께 보는 탐색 흐름
- 13x13 hand grid 중심의 빠른 strategy scanning
- hand 클릭 시 action별 frequency, size, EV detail 표시
- action/size/EV 표시 mode를 바꾸며 같은 solution을 반복 탐색
- source, schema, canonical key, import metadata를 항상 확인할 수 있는 구조

단, GTO Wizard 전체 복제나 solver job generator 구현은 v2.0 범위가 아닙니다. v2.0은 off-table DB browsing foundation입니다.

## Screen Structure

### Browser Tab

- Existing top-level tabs에 Browser 또는 Solution Browser 탭을 추가하는 방향을 계획합니다.
- 첫 구현은 read-only이며 기존 `/api/solutions` 응답을 재사용합니다.
- 별도 API 추가 없이 client-side selection/filter부터 시작합니다.

### Spot Selector

왼쪽 패널은 DB에 실제 존재하는 solution 목록과 spot filter를 제공합니다.

필터 후보:

- hero position
- table size
- remaining players
- stack range
- spot type
- action path
- tree config
- source file
- schema version

규칙:

- DB에 실제 존재하는 값만 옵션으로 표시합니다.
- exact DB 기반 탐색을 유지합니다.
- nearest recommendation을 만들지 않습니다.
- 조건에 맞는 spot이 없으면 빈 상태를 표시합니다.

### Strategy Matrix

가운데 패널은 13x13 strategy matrix를 Browser v2 중심 UI로 표시합니다.

표시 방향:

- hand별 primary action
- mixed action 여부
- action frequency summary
- selected EV mode 값
- missing EV/size/unknown action warning indicator

v2 strategy가 있으면 원본 actions[] 기반으로 표시하고, v1 legacy strategy는 view model fallback을 사용합니다.

### Hand Detail Panel

오른쪽 패널은 선택된 hand의 action detail을 표시합니다.

표시 항목:

- hand
- action kind
- size label
- sizeBb
- sizePctPot
- isAllIn
- rawSizeLabel
- frequency
- EV
- ChipEV
- ICM EV
- sourceActionLabel
- warnings

없는 값은 `제공되지 않음` 또는 `사이즈 미지정`으로 표시합니다. 값을 임의 계산하거나 0으로 채우지 않습니다.

### Filter / Control Bar

Browser 탭 상단 또는 matrix 위에 탐색 control bar를 배치합니다.

Controls:

- action kind filter
- size label filter
- EV display mode: `EV`, `ChipEV`, `ICM EV`
- optional hand search
- optional show mixed only toggle

필터 option은 DB에 실제 존재하는 action/size만 사용합니다.

### Source / Metadata Panel

선택된 solution의 출처와 key 정보를 항상 확인할 수 있게 합니다.

표시 항목:

- source label
- schema version
- canonical key
- source file
- importedAt
- strategy count
- database features
- exact DB 기반 안내
