# poker-tournament-lab v2.3 Plan

## v2.3 Summary

v2.3 plans the first compatibility pass for real HRC export or sanitized real HRC-based sample files.

Unlike v2.2 TEST_ONLY/SAMPLE fixtures, v2.3 focuses on data that was actually produced by HRC, then sanitized for test use. The goal is to verify that import validation, canonical key generation, action tree classification, and Browser display paths remain stable when they see realistic HRC export shape, metadata, action labels, sizes, and EV fields.

The final v2.3 scope separates two tracks:

- Raw HRC export compatibility: `pending_raw_export_required` because no raw HRC export file is currently available.
- Existing imported HRC-derived DB compatibility: available and tested read-only against the current SQLite DB, artifacts, normalized spot JSON, and legacy strategy JSON.

v2.3 does not add a solver, new API, DB schema migration, production DB auto-import, nearest recommendation, RTA/live workflow, or UI implementation.

## v2.2 TEST_ONLY Fixtures vs v2.3 Real HRC Samples

### v2.2 TEST_ONLY / SAMPLE Fixtures

- Purpose: validate UI, classifier, filter, and validator behavior with deliberately small controlled shapes.
- Source: handcrafted test fixtures.
- Metadata: `isSample`, `testOnly`, `TEST_ONLY_SAMPLE`, `MULTI_ACTION_V2_SAMPLE`.
- Interpretation: not real solver output and not real GTO recommendation data.
- Storage: tests only, no production DB change.

### v2.3 Real HRC Sample Fixtures

- Purpose: validate compatibility with actual HRC export shape after sanitization.
- Source: real HRC export or real HRC-based sample file.
- Metadata: `REAL_HRC_SAMPLE` or `HRC_SAMPLE`, `sanitized: true`, `originalTool: HRC`.
- Interpretation: compatibility sample from real HRC output, still not automatically inserted into production DB.
- Storage: sanitized test fixture first, no production DB change unless a later explicit phase approves it.

## Real HRC Sample Intake Principles

Before adding a sample fixture, confirm:

- The sample was generated by HRC or derived from a real HRC export.
- The file is appropriate for repository test use.
- Any private path, user name, player name, machine-specific file path, or local project path can be removed.
- The strategic structure needed for compatibility testing remains intact.
- The sample is not represented as a recommendation or newly computed solution by this app.
- The sample scope is preflop / ICM / MTT where applicable.
- The sample does not introduce RTA/live or poker-client integration semantics.

## Sanitize Rules

Sanitized HRC sample fixtures must remove:

- Personal filesystem paths.
- User names.
- Player names.
- Local machine names.
- Unnecessary export folder names.
- Private tournament identifiers if they are not needed for compatibility.
- Any accidental poker-client, live-session, or screen-capture metadata.

Sanitized fixtures may preserve:

- HRC export shape.
- Spot metadata needed for canonical key construction.
- Action path and tree config.
- Stack, position, table size, and remaining-player metadata.
- Action labels.
- Size labels and raise sizes.
- Frequencies and EV fields needed to test import/display compatibility.
- Source metadata that identifies the file as sanitized HRC sample data.

## Sample Fixture Location Direction

Preferred first locations:

- `packages/core/test/fixtures/real-hrc-sample-fixtures.ts`
- `apps/server/test/fixtures/realHrcSampleFixtures.ts`
- `apps/web/test/fixtures/realHrcSampleFixtures.ts`

Selection principle:

- Put validator/canonical-key compatibility fixtures near core or server tests.
- Put Browser/action tree compatibility fixtures near web tests.
- Avoid runtime seed data.
- Avoid production DB files.
- Avoid changing API response fixtures unless a test explicitly mocks the fixture.

## HRC Sample Metadata Direction

Recommended metadata fields:

- `source`: `HRC_PRECOMPUTED_DB`
- `sampleKind`: `REAL_HRC_SAMPLE` or `HRC_SAMPLE`
- `sanitized`: `true`
- `originalTool`: `HRC`
- `streetScope`: `PREFLOP`
- `calculationModel`: `HRC_EXPORT_SAMPLE` or an existing compatible model label
- `exportShape`: actual observed HRC export shape label
- `sourceFile`: sanitized file name only
- `spotFamily`: Push/Fold, RFI, Limp, Facing Open, Facing Limp, Three Bet, or Unknown when available
- `actionTags`: observed action/tree tags only
- `notes`: short note that this is a sanitized compatibility sample, not production DB content

Metadata rules:

- Do not label real HRC samples as TEST_ONLY handcrafted fixtures.
- Do not label them as newly solved by this application.
- Do not remove source fidelity fields needed to understand compatibility issues.
- Do not add missing action/size values that are not present in the sample.
