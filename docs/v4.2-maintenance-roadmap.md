## Public release upkeep

Every release continues to use a fresh sanitized public folder, an approved
public-safe docs subset, full local verification, normal fast-forward push, and
one explicitly named tag. Force push, `git push --tags`, tag overwrite, and an
original-repo remote remain prohibited.

automatic DB writes, server-side user persistence, or release mutation from
the read-only preflight.
