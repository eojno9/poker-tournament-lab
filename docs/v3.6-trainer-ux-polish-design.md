# v3.6 Trainer UX polish design

## Scope

This document defines the v3.6 Trainer UX polish scope for the Korean-first,
off-table Trainer.

This is a docs-only design step. It does not implement code, change dependencies,
access DB/raw data, create generated artifacts, push to GitHub, or create/change
tags.

## v3.5 Trainer UX Baseline

v3.5 shipped the first Korean-first Trainer UX improvement bundle:

- Korean-first Trainer screen copy
- local-only recent attempts
- local-only mistake queue
- retry, resolved, and dismissed mistake flow
- local summary statistics
- localStorage safe fallback
- visible off-table safety copy

The v3.5 public release is complete with final verdict
`V3_5_PUBLIC_RELEASE_COMPLETE`.

## v3.6 Polish Goals

v3.6 should polish the existing Trainer flow before expanding into deeper
session/statistics work.

Primary goals:

- make the Trainer screen easier to scan
- reduce repeated or overly dense guidance copy
- clarify filter/options grouping
- make the question card hierarchy more obvious
- improve answer feedback copy
- make mistake review entry and empty states clearer
- improve the learning summary card layout
- simplify recent history and local statistics display
- preserve Korean-first user-facing copy
- keep safety copy visible without turning the screen into documentation

## Korean-First UI Criteria

Future implementation should use these criteria:

| Area | Criteria |
| --- | --- |
| Primary labels | Korean-first |
| Buttons | Korean action text, concise and task-specific |
| Feedback | Korean-first, clear correct/incorrect state |
| Empty states | Korean-first, explains what happened and what to do next |
| Technical terms | `EV`, `BB`, canonical identifiers may remain where precise |
| Safety copy | Korean-first, concise, visible, no live-use ambiguity |
| Public data wording | Neutral wording only; no source/provider/permission details |

## Improvement Candidates

### Trainer Top Guidance

Candidate polish:

- shorten the top instructional copy
- keep one visible off-table safety note
- move detailed explanatory text into compact secondary copy
- avoid repeating the same safety sentence in multiple nearby panels

Expected effect:

- easier first read
- less visual weight at the top
- safety boundary remains visible

### Filter And Options Area

Candidate polish:

- group filters by study intent
- make selected filters easier to identify
- clarify reset behavior
- improve empty-result fallback copy
- keep technical filter names only where needed

Expected effect:

- less guessing before starting a session
- fewer confusing empty states

### Question Card Information Structure

Candidate polish:

- make hand/context summary the first visual anchor
- separate question prompt from metadata
- keep candidate answers in a stable layout
- avoid shifting layout during feedback
- keep important technical values compact

Expected effect:

- faster recognition of the current spot
- less distraction while answering

### Correct/Incorrect Feedback Copy

Candidate polish:

- use short Korean state labels for correct and incorrect answers
- explain next action clearly after feedback
- avoid blame-oriented wording
- make retry and mistake queue behavior explicit

Expected effect:

- calmer learning loop
- clearer mistake handling

### Mistake Review Entry And Empty States

Candidate polish:

- make the mistake review entry button visible but not dominant
- show unresolved count clearly
- improve no-mistakes empty state
- improve all-resolved empty state
- reserve deeper queue controls for the mistake review UX step

Expected effect:

- users understand when review is useful
- avoids overbuilding before the dedicated mistake review work

### Learning Summary Card

Candidate polish:

- prioritize accuracy and attempt count
- show mistake count and resolved count compactly
- avoid dense tables unless needed
- handle denominator-0 state safely

Expected effect:

- clearer progress signal after practice
- less noise in early sessions

### Recent History And Statistics

Candidate polish:

- reduce visual dominance of recent attempts
- show a small number of recent entries by default
- keep detailed position/action statistics for a later statistics step
- preserve local-only wording

Expected effect:

- screen stays focused on active practice
- avoids conflicting with the later statistics work

## Implementation Priority

### Must

- top guidance copy cleanup
- visible concise off-table safety copy
- filter/options readability polish
- question card hierarchy polish
- correct/incorrect feedback copy polish
- no-mistakes and all-resolved empty state polish
- learning summary card readability polish
- no package/dependency changes
- no API/server persistence

### Should

- clearer mistake review entry count
- stable answer/feedback layout
- compact recent history presentation
- Korean-first copy audit for the touched Trainer surface
- smoke test updates if user-facing labels change

### Could

- microcopy variants for beginner-friendly explanations
- optional collapsible secondary help text
- small visual grouping changes for statistics
- additional screenshot-based manual review notes

## Relationship To Later v3.6 Work

This polish step should avoid taking over later work:

| Later area | Boundary |
| --- | --- |
| Mistake review UX | Only entry/empty-state polish now; queue mechanics later |
| Session mode | Only current flow clarity now; configured session model later |
| Position/action statistics | Only summary readability now; detailed breakdown later |
| localStorage migration | Do not change storage format in polish step |
| v4.0 readiness | Keep public-safe wording, but full checklist later |

## Non-Goals

This design does not include:

- new DB features
- DB/raw zip/generated report access
- server persistence
- account-based Trainer history
- live/RTA behavior
- OCR, screen capture, overlay, hotkey, watcher, or poker client integration
- new dependencies
- package-lock changes
- product import route changes
- public upload or tag operations

## Acceptance Criteria For Implementation

An implementation step can be considered ready when:

- the Trainer screen remains Korean-first
- top guidance is shorter and easier to scan
- filter/options structure is clearer
- question card hierarchy is visibly improved
- feedback copy is clearer and calmer
- mistake review entry/empty states are improved
- learning summary is easier to read
- no live-use, RTA, OCR, screen capture, overlay, hotkey, watcher, or poker client scope exists
- no DB/raw/generated artifact access exists
- tests/build/smoke pass
