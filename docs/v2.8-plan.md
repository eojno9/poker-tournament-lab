# poker-tournament-lab v2.8 Plan

## v2.8 Summary

v2.8 implements two read-only diagnostic tracks on top of the v2.7 HRC dry-run artifact export infrastructure:

1. A local CLI batch runner that can dry-run multiple repo-external raw HRC zip files and produce sanitized artifact-safe reports, plus index/comparison outputs, only when explicitly allowed.
2. A read-only artifact review dashboard that can inspect existing sanitized `artifacts/hrc-dry-run-reports/*.json` reports without importing, writing to the database, running a solver, or connecting to product import routes.

The completed v2.8 scope includes the explicit batch runner, read-only server artifact API, read-only web dashboard, and integrated scope guard. Product import integration, DB writes, raw zip commits, automatic artifact creation, and solver/live/RTA features remain out of scope.

## Batch Runner Design

Candidate script:

- `scripts/hrcDryRunArtifactBatchExport.ts`

Candidate package script:

- `hrc:dry-run:batch`

Candidate inputs:

- `--input-dir <repo-external-directory>`
- `--zip-list <repo-external-list-file>`
- `--out artifacts/hrc-dry-run-reports`
- `--max-files <number>`
- `--allow-repo-artifact-write`
- `--continue-on-error`
- `--write-index`

High-level flow:

1. Parse explicit CLI arguments.
2. Resolve repo root and output path.
3. Validate that all raw zip inputs are outside the repo.
4. Reject non-zip files, repo-internal zip paths, and path traversal.
5. For each selected zip, run the existing read-only dry-run reader.
6. Convert each dry-run result to the v2.6 artifact-safe report schema.
7. If write is allowed, write sanitized JSON reports under `artifacts/hrc-dry-run-reports`.
8. If requested, generate index/comparison artifacts using existing v2.6/v2.7 helper policy.
9. Summarize successes, warnings, privacy warnings, malformed inputs, and failed entries.

## Batch Runner Safety Policy

- Raw zip paths must be repo-external.
- Repo-internal `.zip` input is rejected.
- Path traversal is rejected.
- Files without `.zip` extension are rejected.
- Raw zip files are never copied into the repo.
- Raw zip files are never fully extracted.
- Only the required settings/selected node data is read in memory through existing dry-run helpers.
- Each report keeps masked path output only.
- Batch failures remain diagnostic and do not become import candidates.
- `--allow-repo-artifact-write` is required for repo artifact writes.
- `--max-files` should provide a conservative safety cap.
- `--continue-on-error` should be explicit so batch failures are visible.

## Artifact Write Policy

- Default behavior is no repo write.
- Repo writes are allowed only with `--allow-repo-artifact-write`.
- Allowed output root is only repo-root-relative `artifacts/hrc-dry-run-reports`.
- Generated report JSON is not automatically committed.
- Generated index/comparison JSON is not automatically committed.
- Raw HRC zip original files are never committed.
- Raw HRC full extraction output is never committed.
- The user must separately decide whether sanitized artifacts should be committed.
