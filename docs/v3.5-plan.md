# v3.5 plan

## Scope

v3.5 focuses on Trainer UX improvements for off-table poker tournament study.

This is a planning-only docs step.

No implementation, dependency change, DB access, raw data access, generated
artifact creation, API integration, UI implementation, GitHub push, or tag
change is performed in this step.

## Safety Constraints

- Off-table study only
- No RTA
- No live assistance
- No OCR
- No screen capture
- No overlay
- No hotkey
- No live watcher
- No poker client integration
- No production DB write
- No raw DB/raw zip access
- No bundled data artifacts
- Users configure their own local data

## Data And Persistence Plan

v3.5 should prefer local-only persistence.

Candidate localStorage keys must be versioned.

Potential areas:

| Data | Persistence | Notes |
| --- | --- | --- |
| Trainer recent attempts | localStorage only | Existing pattern should be reused |
| Mistakes | localStorage only | No server storage |
| Session settings | localStorage only | Optional |
| Accuracy summary | derived locally | Avoid unnecessary persistence |

No server-side user profile, account, or tracking storage should be introduced.

## Implementation Guardrails

Any future implementation must:

- use existing dependencies only unless separately approved
- avoid package-lock changes unless separately approved
- avoid DB/raw/generated artifact access
- avoid API/UI live integration
- keep trainer data local unless explicitly approved
- keep safety wording public-safe and neutral
- avoid live-game workflows
- avoid poker-client integration
- preserve deterministic/off-table behavior
