# poker-tournament-lab v1.7 Plan

## v1.7 Summary

v1.7 builds the read-only foundation for a multi-action strategy shape. The goal is to move from a legacy `hand -> action/frequency/EV` view toward a backward-compatible `hand -> actions[]` model that can later support GTO Wizard-style solution browsing.

This release does not add a solver, does not migrate the database schema, and does not change exact lookup or fallback routing.

## v1.6 Stable Baseline

- Stable tag: `v1.6`
- Stable commit: `c91dc7caf8e083d4de81fd47eb43fe760afd7b77`
- v1.6 backup restore rehearsal: PASS
- Expected starting state: clean working tree

## Product Goals

- Short term: Import HRC-built DBs accurately and explore/study them like GTO Wizard.
- Mid term: Let users choose only DB-existing actions/raise sizes, then display raise/call/fold/all-in frequencies and EVs clearly.
- Long term: Create calculation jobs like HRC Pro / ICMIZER and keep producing GTO DBs from internal or external calculation results.
- Final goal: Build an off-table MTT preflop/ICM GTO DB creation, exploration, and study platform.

## Goals

- Design `hand -> actions[]` multi-action schema v2 types.
- Add backward-compatible adapters for existing single-action strategies.
- Prepare a data foundation for raise/call/fold/all-in frequency and EV display.
- Add read-only foundations for a GTO Wizard-style Solution Browser v2.
- Preserve v1.6 Analyze, Import, Database, Trainer, Verification, and action/sizing selector behavior.

## Non-Goals

- No DB schema migration.
- No new solver.
- No Nash or approximate Nash solver.
- No PKO, bounty, satellite, or postflop logic.
- No nearest recommendation.
- No OCR, screen capture, overlay, hotkey, live watcher, poker client integration, real-time assistance, or RTA workflow.
- No tag or backup creation during v1.7 implementation.
