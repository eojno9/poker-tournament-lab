# v3.5 Trainer session model design

## Scope

This document defines the planned Trainer session model for v3.5.

This is a docs-only design step.

No implementation, dependency change, DB access, raw data access, generated
artifact creation, API integration, UI implementation, GitHub push, or tag
change is performed in this step.

## Safety Constraints

- Off-table study only
- No RTA
- No live assistance
- No OCR
- No screen capture
- No overlay
- No hotkey
- No live watcher
- No poker client integration
- No production DB write
- No raw DB/raw zip access
- No bundled data artifacts
- Local-only Trainer persistence

## Design Goals

| Goal | Description |
| --- | --- |
| Session clarity | Make a Trainer session feel like a clear start-progress-finish flow |
| Local-only persistence | Keep Trainer session/history data in localStorage only |
| Mistake review compatibility | Store enough local metadata to support later mistake retry flow |
| Filter compatibility | Preserve selected position/table/tree/source filters in session state |
| Beginner-friendly summary | Provide clear end-of-session stats without implying live advice |
| Fail-safe reset | Allow local reset without server involvement |

## Session Lifecycle

Define the planned lifecycle:

1. Not started
2. Configured
3. In progress
4. Answer submitted
5. Feedback shown
6. Next question
7. Completed
8. Reset / cleared

## Session State Model

Candidate shape:

```ts
type TrainerSessionState = {
  version: 1;
  sessionId: string;
  createdAt: string;
  updatedAt: string;
  status: "not_started" | "configured" | "in_progress" | "completed";
  filters: TrainerSessionFilters;
  questionIds: string[];
  currentIndex: number;
  attempts: TrainerSessionAttempt[];
  summary: TrainerSessionSummary;
};
```

This is design-only and must not be implemented in this step.

## Filter Model

Candidate shape:

```ts
type TrainerSessionFilters = {
  position?: string;
  tableSize?: string;
  treeType?: string;
  source?: "LOCAL_PRECOMPUTED_STUDY_DATA";
  handSearch?: string;
  seed?: string;
};
```

Notes:

- Keep source wording public-safe and neutral.
- Do not introduce live or real-time data sources.
- Empty filters should be treated as broad off-table study filters.
- Avoid public wording that names a specific external dataset, provider,
  permission source, or original file path.

## Attempt Model

Candidate shape:

```ts
type TrainerSessionAttempt = {
  attemptId: string;
  questionId: string;
  solutionId: string;
  hand: string;
  position?: string;
  selectedAction: "SHOVE" | "FOLD" | "CALL" | "RAISE" | "OTHER";
  correctAction?: string;
  isCorrect: boolean;
  answeredAt: string;
  elapsedMs?: number;
  source: "LOCAL_PRECOMPUTED_STUDY_DATA";
  reviewStatus?: "new" | "retry" | "resolved";
};
```

Notes:

- This is off-table study feedback only.
- Do not include any live hand import, OCR, screen capture, or poker-client
  metadata.
- Do not store original raw data filenames or local source paths in attempts.

## Summary Model

Candidate shape:

```ts
type TrainerSessionStat = {
  attempted: number;
  correct: number;
  incorrect: number;
  accuracyPct: number;
};

type TrainerSessionSummary = {
  totalQuestions: number;
  attempted: number;
  correct: number;
  incorrect: number;
  skipped: number;
  accuracyPct: number;
  byPosition?: Record<string, TrainerSessionStat>;
  byAction?: Record<string, TrainerSessionStat>;
};
```

## localStorage Plan

Candidate keys:

| Key | Purpose | Version |
| --- | --- | --- |
| `ptl.trainer.session.v1` | Current active local Trainer session | v1 |
| `ptl.trainer.recentAttempts.v1` | Recent local attempts | v1 |
| `ptl.trainer.mistakes.v1` | Local mistake queue | v1 |
| `ptl.trainer.sessionSettings.v1` | Optional remembered filters/settings | v1 |

Rules:

- localStorage only
- no server persistence
- no user account tracking
- no private synced storage
- safe fallback if parsing fails
- clear/reset action should be available

## State Transitions

| From | Event | To | Notes |
| --- | --- | --- | --- |
| not_started | configure session | configured | Filters selected |
| configured | start session | in_progress | Question list fixed |
| in_progress | submit answer | in_progress | Feedback shown |
| in_progress | next question | in_progress | Advance index |
| in_progress | finish final question | completed | Summary generated |
| completed | restart | configured | Keep filters if user chooses |
| any | reset local session | not_started | Clear current session only |

## Derived Statistics

Define derived local stats:

| Stat | Formula / rule |
| --- | --- |
| attempted | count submitted attempts |
| correct | count `isCorrect = true` |
| incorrect | count `isCorrect = false` |
| skipped | count skipped questions, if implemented |
| accuracyPct | correct / attempted, rounded safely |
| byPosition | grouped local attempts by position |
| byAction | grouped local attempts by selected action |

Avoid misleading precision. Label all stats as local Trainer history only.

## Empty And Unsupported States

Define UX handling:

| State | Message intent |
| --- | --- |
| No matching questions | Explain filters returned no local off-table study spots |
| Unsupported source | Explain Trainer supports precomputed off-table study spots only |
| Corrupt localStorage | Safely reset local state and show non-alarming message |
| No attempts yet | Show beginner-friendly empty state |
| No mistakes yet | Explain mistake review appears after incorrect attempts |

## Safety Copy Requirements

Future UI copy must include:

- off-table study only
- no live assistance
- no poker client integration
- locally configured precomputed study data
- local-only Trainer history

Do not include:

- real-time recommendation wording
- live hand monitoring wording
- OCR/screen capture wording
- poker client integration wording

## Future Implementation Guardrails

Any later implementation must:

- use existing dependencies unless separately approved
- avoid package-lock changes unless separately approved
- avoid DB/raw/generated artifact access
- avoid API/UI live integration
- preserve local-only persistence
- add tests for localStorage parse failure
- add tests for summary calculations
- add tests for reset behavior
