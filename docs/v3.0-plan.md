# poker-tournament-lab v3.0 Product Import Design Plan

## Goal

v3.0 designs the product DB import path for real HRC-derived data that was rehearsed in v2.9. The design starts from the real HRC dry-run results and import candidate classification, but this initial v3.0 step does not write to the database, change schema, connect product import routes, or add UI controls.

The first objective is to define the data contract, safety gates, approval policy, and rollback plan that must exist before any product import implementation can be considered.

## v2.9 Inputs

v2.9 produced the review materials that v3.0 will use as design inputs:

- Inventory report: `docs/v2.9-real-hrc-inventory-report.md`
- Controlled batch dry-run report: `docs/v2.9-controlled-batch-dry-run-report.md`
- Artifact safety/dashboard review: `docs/v2.9-artifact-safety-dashboard-review.md`
- Import candidate classification: `docs/v2.9-import-candidate-classification.md`

Classification summary:

- `IMPORT_CANDIDATE`: 19
- `NEEDS_MANUAL_REVIEW`: 8
- `HOLD`: 0
- `EXCLUDE`: 1

v2.9 confirmed that raw HRC zip files were not committed, generated dry-run artifact JSON was not committed, privacy/path scans passed, dashboard review passed, product import routes remained disconnected, and no production DB write was applied.

## Scope

v3.0 will evaluate and design:

- Data contract for converting the 19 `IMPORT_CANDIDATE` entries into product DB import candidates.
- Hold/review policy for the 8 `NEEDS_MANUAL_REVIEW` entries.
- Exclusion policy for the 1 `EXCLUDE` entry.
- Import preview and dry-run policy before any write-capable flow exists.
- Source provenance policy for HRC-derived data.
- Duplicate and canonical key conflict policy.
- Backup and rollback policy before any import write.
- Explicit approval gate required before DB write.
- Import report generation policy.
- Safety conditions required before any API or UI connection.

## Non-Goals

This plan step explicitly does not include:

- DB write.
- DB schema migration.
- Product import route connection.
- UI import button or workflow.
- API changes.
- Raw HRC zip commit.
- Raw HRC zip copy or extraction into the repo.
- Generated artifact JSON commit.
- `artifacts/hrc-dry-run-reports` creation or commit.
- Live/RTA/OCR/overlay/hotkey/poker client integration.
- Nash, PKO, bounty, or postflop solver implementation.
