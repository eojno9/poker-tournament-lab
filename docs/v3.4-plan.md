## Dry-run-only skeleton summary

The command skeleton parses a constrained flag set:

- `--dry-run`
- `--config <path>`
- `--target-kind copied-local`
- `--approval <value>`
- `--console-only`

The parser treats config paths as strings only. It does not read config files.

## Safety gate summary

The no-write safety gate returns `DRY_RUN_PASS` only when all required dry-run conditions are present.

It fails closed for:

- missing `--dry-run`
- missing config
- missing `--console-only`
- target kind other than `copied-local`
- missing local rehearsal approval marker
- production-like inputs
- raw DB or archive-like inputs
- write/import/apply/migrate/commit flags
- API/UI flags
- live/OCR/screen-capture/overlay/hotkey/watcher/poker-client flags
- output-file/report-file flags
- unknown flags or positional arguments

## Report renderer summary

The renderer returns console-ready text only.

Required output includes:

- `Mode: dry-run`
- `Write status: no-write`
- `Raw data status: not accessed`
- `Production DB status: not targeted`
- `Scope: off-table study only`
- `Verdict: DRY_RUN_PASS` or `Verdict: DRY_RUN_BLOCKED`

The renderer does not print concrete user config paths, data source details, private paths, or raw file names.

## Test summary

The unit tests cover:

- missing dry-run flag
- missing config
- missing console-only mode
- wrong target kind
- missing local rehearsal approval marker
- production-like input
- raw DB-like input
- raw archive-like input
- write-like flags
- API/UI flags
- live-scope flags
- report-output flags
- valid dry-run pass
- public-safe renderer output
- CLI non-zero exit shape for unsafe input

No DB, raw archive, generated artifact, API/UI integration, or package script fixture is added.
