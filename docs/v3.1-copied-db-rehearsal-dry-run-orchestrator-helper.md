# v3.1 Copied DB Rehearsal Dry-run Orchestrator Helper

## Purpose

The copied DB rehearsal dry-run orchestrator helper composes the v3.1 copied DB rehearsal pure helpers into one deterministic dry-run result shape.

The composition flow is:

1. copied DB target approval contract
2. copied DB rehearsal guard
3. copied DB rehearsal plan builder
4. copied DB rehearsal report shape

The orchestrator accepts only in-memory input and returns only in-memory output.

## Non-goals

- No actual DB copy.
- No actual DB write.
- No production DB write.
- No DB schema migration.
- No product import route connection.
- No API/UI import flow.
- No `package.json` script.
- No raw HRC zip access.
- No report JSON file generation.
- No filesystem access.
- No DB access.
- No `process.env`, `Date.now`, or random value usage.

## Inputs

The helper accepts:

- rehearsal id
- target summary and copied DB path guard result
- approval request
- validation summary
- DB SHA summary
- safety inputs
- execution summary
- report policy

All inputs must already be safe, redacted, or test-only placeholders.

## Output Shape

The output includes:

- approval contract
- rehearsal guard result
- rehearsal plan
- rehearsal report
- compact summary
- always-false production/report write flags
- always-false orchestrator execution flags

The orchestrator itself does not perform writes and records:

- `orchestratorDbCopyPerformed: false`
- `orchestratorDbWritePerformed: false`
- `orchestratorReportJsonWritten: false`

## Status and Exit Code

The final status and exit code come from the report helper:

- `READY_FOR_DRY_RUN` with exit code `0`
- `READY_FOR_COPIED_DB_WRITE_REHEARSAL` with exit code `0`
- `BLOCKED` with exit code `1`, `2`, `3`, or `4`

Exit code meanings:

- `0`: ready dry-run result shape
- `1`: validation or plan blocked
- `2`: safety blocked
- `3`: privacy/path blocked
- `4`: internal/input shape blocked

## Blocked Conditions

The orchestrator propagates blocks from every composed helper, including:

- production DB target
- repo-local DB target
- missing approval
- invalid approval
- privacy scan failure
- raw zip presence
- actual DB write evidence
- report JSON write evidence
- original DB SHA mismatch
- duplicate validation failure
- missing rollback plan

## Safety Invariants

- `productionDbWriteAllowed` is always `false`.
- `reportFileWriteAllowed` is always `false`.
- No actual DB copy is performed.
- No actual DB write is performed.
- No report JSON is written.
- Product import route remains disconnected.
- API/UI/runtime import flow remains unchanged.
- Raw HRC zip access remains out of scope.
