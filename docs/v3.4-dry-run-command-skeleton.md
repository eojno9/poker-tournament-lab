# v3.4 dry-run command skeleton

## Implementation purpose

v3.4 adds a minimal dry-run-only copied DB rehearsal command skeleton.

The skeleton is not wired into `package.json`. It is a source-level entry candidate that can be reviewed before any package script or broader workflow is approved.

## Implemented files

| File | Purpose |
| --- | --- |
| `apps/server/src/copiedDbRehearsalConfig.ts` | Pure argument parser and value redaction |
| `apps/server/src/copiedDbSafetyGate.ts` | Pure fail-closed safety gate |
| `apps/server/src/copiedDbRehearsalReport.ts` | Public-safe console report renderer |
| `apps/server/src/cli/copiedDbRehearsal.ts` | CLI entry skeleton returning exit code and console-ready text |
| `apps/server/test/copiedDbRehearsal.test.ts` | Pure unit tests for parser, gate, renderer, and CLI shape |

## Parser contract

The parser accepts only:

- `--dry-run`
- `--config <path>`
- `--target-kind copied-local`
- `--approval <value>`
- `--console-only`

The parser does not read files. It records values as strings and lets the safety gate classify unsafe inputs.

Forbidden flags are classified and passed to the gate without executing any behavior.

## Safety gate contract

The safety gate returns:

- `DRY_RUN_PASS`
- `DRY_RUN_BLOCKED`

It includes:

- `verdict`
- `allowed`
- `reasons`
- `mode`
- `writeStatus`
- `rawDataStatus`
- `productionDbStatus`
- `scopeStatus`

The safety gate always reports:

- `dbWriteAllowed: false`
- `productionDbWriteAllowed: false`
- `reportFileWriteAllowed: false`

## Renderer contract

The renderer returns a single string suitable for console output.

It prints only public-safe category status:

- dry-run mode
- no-write status
- raw data not accessed
- production DB not targeted
- off-table-only scope
- final verdict
- public-safe refusal reasons

It does not print concrete user paths, raw file names, data source details, provider names, or permission claims.

## CLI entry skeleton

`runCopiedDbRehearsalCli(argv, io)` parses args, evaluates the safety gate, renders the report, and returns:

- exit code `0` for safe dry-run shape
- exit code `1` for blocked input
- stdout lines
- stderr refusal lines for blocked input
- parsed args and safety gate result for tests/review

The optional `io` dependency can receive console-ready strings. The module has no package script and no top-level process exit.
