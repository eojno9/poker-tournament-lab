# v3.0 Copied DB Path Guard Pure Helper

## Purpose

The copied DB path guard is a pure string helper for future copied-DB-only rehearsal or no-write CLI skeleton work. Its purpose is to reject production DB targets, repo-local targets, SQLite sidecar files, raw HRC paths, and unsafe path exposure before any future rehearsal command can consider a target DB path.

This helper does not copy a DB, create a copied DB, read or write files, access the DB, implement a command, add a package script, call APIs, change UI, or connect product import routes.

## Non-Goals

- No actual DB copy.
- No copied DB creation.
- No rehearsal execution.
- No filesystem existence checks.
- No DB read or write.
- No import command implementation.
- No package script.
- No API endpoint.
- No UI workflow.
- No product import route.
- No schema migration.
- No report JSON generation.
- No raw HRC zip or external artifact access.

## Input Shape

`HrcCopiedDbPathGuardInput` contains only strings and an optional placeholder policy:

- `targetDbPath`
- `productionDbPath`
- `repoRootPath`
- `backupRootPath`
- `allowPlaceholderPathsOnly`

The helper treats these as plain strings. It does not check whether any path exists.

## Output Shape

`HrcCopiedDbPathGuardResult` returns:

- `allowed`
- `decision`
- `normalizedTargetDbPathRedacted`
- `normalizedProductionDbPathRedacted`
- `reasons`
- `warnings`

Outputs must be safe for reports. Local paths, raw HRC path markers, user tokens, and email-like values are redacted.

## Decisions

- `ALLOWED_COPIED_DB_TARGET`: target is a `.db` file under the approved backup root.
- `BLOCKED_PRODUCTION_DB_TARGET`: target matches the production DB path after normalization.
- `BLOCKED_REPO_LOCAL_TARGET`: target is under the repo root.
- `BLOCKED_RAW_HRC_PATH`: target contains raw HRC folder or raw zip-style markers.
- `BLOCKED_PRIVATE_PATH_EXPOSURE`: placeholder-only mode is enabled and target contains private local path markers.
- `BLOCKED_INVALID_EXTENSION`: target does not end in `.db`.
- `BLOCKED_EMPTY_PATH`: target is empty after trimming.
- `BLOCKED_SIDE_CAR_FILE`: target points to `.db-shm` or `.db-wal`.
- `BLOCKED_UNKNOWN`: target is not under the approved copied DB backup root.

## Normalization Policy

The comparison helper normalizes slash style, trims surrounding whitespace, removes trailing slashes, and uses case-insensitive comparison for path containment checks. This allows Windows slash/backslash variations to resolve to the same comparison form without touching the filesystem.

## Production DB Target Blocking

If `targetDbPath` and `productionDbPath` normalize to the same comparison value, the result is `BLOCKED_PRODUCTION_DB_TARGET`. This check is performed before repo-local and extension checks so the production DB cannot be misclassified as a generic repo-local path.

## Repo-local Target Blocking

If `targetDbPath` is under `repoRootPath`, the result is `BLOCKED_REPO_LOCAL_TARGET`. Future copied DB rehearsal targets must live outside the repo.

## Sidecar File Blocking

SQLite sidecar files are not valid copied DB targets. The guard blocks:

- `.db-shm`
- `.db-wal`

## Raw And Private Path Policy

The guard blocks raw HRC path markers and redacts local/private tokens from output. Forbidden output includes:

- local absolute path prefixes
- user tokens
- private relay email-like values
- raw HRC folder markers
- raw HRC zip path markers

When `allowPlaceholderPathsOnly` is true, private local path markers in the target are blocked rather than merely redacted.
