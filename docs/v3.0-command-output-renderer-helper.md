# v3.0 Command Output Renderer Pure Helper

## Purpose

This step defines the safe console-output shape for a future no-write HRC import CLI command as a pure core helper. The renderer converts an in-memory command report and optional no-write command plan into human-readable lines and text.

This helper does not print to the console, implement a CLI entrypoint, add a package script, write files, generate report JSON, read or write the DB, call APIs, change UI, or connect product import routes.

## Non-Goals

- No actual console output.
- No actual CLI entrypoint.
- No package script.
- No direct process argument handling.
- No file write.
- No JSON report generation.
- No DB read or write.
- No DB copy or copied DB creation.
- No rehearsal execution.
- No API, UI, server runtime, or product import route connection.
- No raw HRC zip or external artifact access.

## Input Shape

`HrcImportCommandOutputRenderInput` accepts:

- `commandReport`
- `commandPlan`
- `includeWarnings`
- `includeNextAction`

The command report is expected to come from the existing command report shape helper. The command plan is optional and can come from the no-write CLI command skeleton helper.

## Output Shape

`HrcImportCommandOutputRenderResult` returns:

- `lines`
- `text`
- `lineCount`
- `hasWarnings`
- `forbiddenExposureDetected`
- `warnings`

`text` is always derived from `lines.join("\n")`. The renderer returns data only.

## Output Sections

The renderer produces these sections:

- Header: command, mode, status, exit code, and write flags.
- Preview Summary: total, import preview allowed, manual review required, and excluded counts.
- Validation Summary: duplicate existing DB, duplicate in batch, missing canonical key, privacy blocked, and blocking issue counts.
- Safety Gates: DB SHA256 unchanged, product import route disabled, DB read/write performed, and write flags.
- Warnings: optional.
- Next Action: optional.

## Redaction And Forbidden Exposure Policy

The renderer redacts local paths, user tokens, raw HRC markers, and email-like values from rendered output. If the raw render input would have exposed a forbidden value, `forbiddenExposureDetected` is set to `true` and a renderer warning is added.

Rendered text must not expose:

- local absolute paths
- user tokens
- private relay or email-like values
- raw HRC folder markers
- raw HRC zip path markers

## Write Flag Display Policy

The renderer explicitly displays:

- `Write Allowed: false`
- `DB Write Allowed: false`
- `Report File Write Allowed: false`

When a no-write command plan is provided, it also displays:

- `Dry Run Only: true`
- `Force Allowed: false`

## Future Step Candidates

Any future command entrypoint must remain no-write by default and must not connect to product import routes without a separate explicit approval step.
