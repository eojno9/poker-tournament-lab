# v3.9 scope audit

## Scope

v3.9 is the final stabilization audit before v4.0. The goal is not to add a
large new feature, but to identify practical user-flow, readiness, and quality
gaps that are worth fixing before the v4.0 planning step.

This audit is docs-only. It does not change code, package files, DB/raw data,
generated artifacts, local tags, GitHub refs, or remotes.

## Investigation Targets

Reviewed areas:

- Analyze input, validation, presets, and recent analysis flow
- Database search, detail view, and Database to Analyze handoff
- Trainer session, filters, mistake review, local statistics, and storage
- canonical-key and verification-facing explanations
- screen-to-screen movement and state retention
- empty, loading, error, invalid, and not-solved states
- Korean-first visible copy and accessible names
- keyboard and focus handling
- narrow-screen and long-text layout expectations
- localStorage keys, migration, clear, reset, and write-failure behavior
- README, SECURITY, CONTRIBUTING, RELEASE_HISTORY, and CI workflow
- core/server/web/smoke test coverage
- sanitized public upload workflow
