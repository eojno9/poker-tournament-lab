# v4.2 scope audit

## Baseline

v4.1 is the completed public maintenance release. Its verified baseline is:

- original tag target: `b684054846ae73b71c5a7f603034fb7cfcdb96ed`
- public snapshot/main: `b695fc83d0667b6abddde8e8636875f40526807b`

- tests: core `383`, server `82`, web `130`
- smoke: `7/7`

v4.1 extracted `TrainerView`, added failure-safe storage I/O, mapped browser API
failures to Korean-first public messages, and added a read-only release
preflight. v4.2 evaluates the remaining maintenance gaps without adding a new
product capability.

## Already resolved

- Trainer state and UI ownership are outside `App.tsx`.
- Browser storage availability and I/O exceptions share one safe boundary.
- Domain modules retain parsing, migration, limits, and reset semantics.
- Browser API failures do not render raw response bodies.
- Storage keys and versions are documented and tested.
- CI uses Node.js 24 and current approved official action versions.
- The public release preflight is read-only in its current original mode.
- The v4.1 release avoided a post-tag status hotfix by using time-stable status
  wording before the snapshot commit.

## No implementation items

- No storage key rename or version bump is justified.
- No storage parsing framework is justified; domain validators are different
  enough that a generic parser would weaken contracts.
- No dependency change is justified.
- No warning suppression is justified for the Node SQLite experimental notice.
- No new solver, data bundle, persistence service, account system, or live
  integration is in scope.

## User-visible behavior

The intended v4.2 maintenance work preserves user behavior. The only visible
changes permitted are safer generic server failures, corrected public status
copy, and test-supported responsive behavior. HRC extraction, test helpers,
and preflight modes are structural or release-maintenance changes.

## Scope verdict

`V4_2_SCOPE_CONFIRMED`

The work is maintenance-only and has concrete safety and maintainability
value. A focused implementation bundle is justified.
