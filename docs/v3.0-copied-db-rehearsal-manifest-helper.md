# v3.0 Copied DB Rehearsal Manifest Helper

## Purpose

This step defines a pure manifest shape for future copied-DB-only rehearsal results.

The helper records the expected metadata and safety state for a copied DB rehearsal preview. It does not create a copied DB, execute a rehearsal, read or write a DB, write files, generate report JSON, call APIs, change UI/runtime behavior, or connect the product import route.

## Non-goals

- Not actual copied DB creation.
- Not an actual rehearsal.
- Not DB read/write.
- Not DB schema migration.
- Not file write.
- Not report JSON generation.
- Not API/UI/runtime connection.
- Not product import route connection.
- Not raw HRC zip access.
- Not external artifact JSON access.

## Manifest Fields

The manifest includes:

- `version`
- `rehearsalId`
- `timestampIso`
- `branchName`
- `commitHash`
- `productionDbSha256Before`
- `productionDbSha256After`
- `productionDbSha256Unchanged`
- `copiedDbSha256Before`
- `copiedDbSha256After`
- `copiedDbSha256Changed`
- `copiedDbPathGuardDecision`
- `dryRunExitCode`
- `importPreviewAllowed`
- `validationSummary`
- `safetyChecks`
- `rollbackPlanAvailable`
- `explicitApprovalRecorded`
- `productionDbWriteAllowed`
- `copiedDbWriteAllowed`
- `reportFileWriteAllowed`
- `warnings`

All timestamp, ID, branch, commit, SHA, and summary values are supplied by the caller. The helper does not call `Date.now`, generate random IDs, inspect files, or query a database.

## Production DB SHA Policy

`productionDbSha256Before` and `productionDbSha256After` are compared as input strings.

- If they match, `productionDbSha256Unchanged` is `true`.
- If they differ, `productionDbSha256Unchanged` is `false` and the manifest records a warning.
- The helper never reads the production DB to compute SHA values.

## Copied DB SHA Policy

`copiedDbSha256Before` and `copiedDbSha256After` are also caller-supplied strings.

- If they match, `copiedDbSha256Changed` is `false`.
- If they differ, `copiedDbSha256Changed` is `true`.
- Even if the copied DB SHA changes in a future rehearsal record, this Step keeps all write flags false.
- The helper never creates, opens, copies, reads, or writes the copied DB.

## Write Flag Policy

The helper always returns:

- `productionDbWriteAllowed: false`
- `copiedDbWriteAllowed: false`
- `reportFileWriteAllowed: false`

This remains true even when `explicitApprovalRecorded` is `true`. Any future write-capable rehearsal requires a separate approved step.

## Safety Checks

The manifest records these safety checks:

- `gitStatusClean`
- `testPassed`
- `buildPassed`
- `smokePassed`
- `productionDbShaUnchanged`
- `copiedDbTargetAllowed`
- `productImportRouteDisabled`
- `dbReadWriteNotPerformed`
- `reportJsonNotGenerated`
- `rawZipAbsent`
- `generatedArtifactJsonAbsent`
- `hrcDryRunReportsAbsent`
- `privacyPathScanPassed`

`validateHrcCopiedDbRehearsalSafetyChecks` reports failed checks and warning messages. A false safety check never enables write behavior.

## Rollback Plan Policy

`rollbackPlanAvailable` is a caller-supplied boolean.

- If `true`, the manifest records that a rollback plan is available.
- If `false`, the manifest records a warning.
- The helper does not execute rollback, restore rehearsal, file copy, or DB restore operations.

## Explicit Approval Policy

`explicitApprovalRecorded` is recorded for future audit shape only.

In this Step:

- It does not enable production DB writes.
- It does not enable copied DB writes.
- It does not enable report file writes.
- It does not run a rehearsal.

## Privacy And Path Redaction

Manifest output redacts local/private/raw path tokens from string fields and JSON summary values.

The output must not expose:

- raw absolute local paths
- user tokens
- private relay emails
- sample-external-hrc-folder paths
- raw HRC source folder paths

## Helper API

Pure functions:

- `buildHrcCopiedDbRehearsalManifest(input)`
- `validateHrcCopiedDbRehearsalSafetyChecks(manifest)`
- `summarizeHrcCopiedDbRehearsalManifest(manifest)`

These functions are in-memory only and do not perform filesystem, DB, API, runtime, or report generation operations.
