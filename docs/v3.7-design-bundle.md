## Session Flow Design

### Current State

v3.6 shows a lightweight local session card with current problem, session attempts,
session accuracy, and session start time. Reset clears current session counters and
current answer state.

### v3.7 Target

The Trainer should make session state easier to understand:

- not started
- in progress
- completed
- reset

The UI should remain compact. It should not add a wizard or account-like workflow.

Recommended local-only fields:

| Field | Purpose |
| --- | --- |
| `sessionStatus` | `not_started`, `in_progress`, `completed` |
| `sessionGoalCount` | optional local target count, default from current candidates if omitted |
| `sessionCompletedAt` | completion timestamp for local display only |
| `sessionFilterSnapshot` | copy of the local filter summary used for this session |

Korean-first labels:

- `세션 시작`
- `세션 진행 중`
- `세션 완료`
- `세션 다시 시작`
- `이번 세션 요약`

## Filter Settings Save/Restore Design

### Goal

Allow users to keep a local Trainer filter setup without creating server
persistence.

### Storage Policy

Use a versioned localStorage key:

- `ptl.trainer.filters.v1`

Stored data should be limited to current Trainer filter fields:

- hero position
- table size
- tree config
- local source label/value
- hand text
- seed text

Do not store:

- raw paths
- private data source details
- DB file names
- account/user identifiers
- remote URLs
- raw imported payloads

### Fallback Rules

- missing value: use current defaults
- invalid JSON: reset to defaults if possible
- unknown version: ignore and preserve default UI
- partial payload: merge valid known fields only
- storage write failure: keep current UI usable

Korean-first labels:

- `필터 저장`
- `저장한 필터 불러오기`
- `저장한 필터 초기화`
- `저장된 필터가 없습니다`

## Mistake Review Sorting And Filtering Design

### Goal

Make the local mistake queue easier to scan without changing the data model
more than necessary.

Must support:

- unresolved first
- resolved view
- hidden view
- newest first within each state

Recommended filter modes:

- `미해결`
- `해결됨`
- `숨김`
- `전체`

Recommended sort modes:

- `최근 오답순`
- `최근 복습순`
- `재시도 많은 순`

The first implementation can start with filter modes only if sort modes make the
UI too dense.

## Local Statistics Range Design

### Goal

Clarify whether numbers represent all local history, current session, or recent
attempts.

Must preserve:

- denominator-0 safe display
- local-only wording
- position/action buckets

Should add:

- `전체 로컬 기록`
- `최근 10문제`
- `이번 세션`

Avoid adding charts unless a later step separately designs them.

## localStorage Migration Design

v3.7 should introduce migration helpers only where implementation touches storage.

Versioned keys:

- `ptl.trainer.recent.v1`
- `ptl.trainer.mistakes.v1`
- `ptl.trainer.filters.v1`
- optional later: `ptl.trainer.sessions.v1`

Migration rules:

- never throw during UI initialization
- never block Trainer rendering because of a corrupt local payload
- reset corrupt v1 payloads to default when safe
- ignore unknown future versions
- keep tests for corrupt, missing, partial, and write-failure cases

## Accessibility And Keyboard Baseline

v3.7 should check:

- buttons have readable accessible names
- segmented/filter controls can be reached by keyboard
- focus order follows the visual flow
- destructive local clear actions are visibly distinct
- narrow viewport wrapping does not hide text
- dynamic counters do not shift layout excessively
- empty states and summaries remain readable without hover

This is a baseline pass, not a full accessibility certification.

## Responsive Layout Baseline

Review these areas at narrow widths:

- top Trainer guidance
- filter controls
- session card
- problem card
- action buttons
- summary strip
- mistake review filters/list rows

Use stable grid/flex sizing and avoid text overflow in buttons or compact status
blocks.
