## Package script approval criteria

A future package script connection is acceptable only if all of the following remain true:

- the command remains dry-run-only
- the script name clearly communicates rehearsal and dry-run behavior
- no DB write path is introduced
- no raw DB or raw archive access is introduced
- no generated report file is written by default
- no API route, Web UI, or runtime app flow connection is introduced
- no live assistance, RTA, OCR, screen capture, overlay, hotkey, watcher, or poker-client integration is introduced
- tests remain green
- the `package.json` change is explicitly approved in a separate step
- `package-lock.json` and dependencies remain unchanged unless separately approved

Candidate script shape for a future approved step:

```json
{
  "scripts": {
    "copied-db:rehearsal:dry-run": "node --import tsx --eval \"...\" --"
  }
}
```

This candidate was implemented in the follow-up connection step after approval. The script name keeps the dry-run boundary explicit and uses the existing `tsx` runtime loader without adding dependencies.
