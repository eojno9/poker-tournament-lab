## Implementation Summary

v3.8 focused on stability and readiness rather than adding a broad new feature.

Implemented results:

- Trainer state transitions guard empty candidate lists, completed sessions,
  reset actions, context/filter changes, retry paths, and duplicate answer
  submission.
- Trainer storage handling is consolidated around the current versioned local
  keys: `ptl.trainer.recent.v1`, `ptl.trainer.mistakes.v1`, and
  `ptl.trainer.filters.v1`.
- Recent-history legacy migration remains compatible with the previous
  `ptl.trainer.recentAttempts.v1` key.
- Corrupt, partial, unknown, unavailable, reset-failure, and write-failure
  storage cases fall back safely without stopping Trainer use.
- Trainer controls have clearer accessible names, keyboard-reachable selected
  states, focus-visible styling, disabled-state feedback, and semantic status
  regions.
- Narrow/mobile layout handling is improved for Trainer guidance, filters,
  question cards, action buttons, session summaries, mistake review, and local
  statistics.
- README public-status wording is neutral for the latest sanitized public
  snapshot, and SECURITY reporting guidance uses public-safe wording.
