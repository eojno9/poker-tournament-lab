## Reviewed Steps

### Step 1: Copied DB Import Rehearsal Design

Documented a copied-DB-only rehearsal process on top of the v3.0 no-write foundation. It defined copied DB creation policy, SHA baseline evidence, approval gates, rollback verification, and post-rehearsal reporting boundaries.

### Step 2: Copied DB Target Approval Contract Helper

Added a pure approval contract helper for future copied DB rehearsal targets. It evaluates explicit approval evidence and target guard results, but it never performs writes. Output write flags remain false.

### Step 3: Copied DB Rehearsal Guard Helper

Added a pure guard helper that evaluates target kind, target location, approval evidence, SHA evidence, preview validation, duplicate validation, rollback evidence, backup evidence, privacy scan, and source archive DB injection policy acknowledgment.

### Step 4: Copied DB Rehearsal Plan Builder Helper

Added a pure plan builder that turns approval and guard evidence into `READY_FOR_DRY_RUN`, `READY_FOR_COPIED_DB_WRITE_REHEARSAL`, or `BLOCKED` plan shapes.

### Step 5: Copied DB Rehearsal Report Shape Helper

Added a pure report shape helper that turns plan, count, SHA, safety, and execution evidence into an in-memory report and exit code. It blocks evidence of actual DB copy, actual DB write, and report JSON writes.

### Step 6: Copied DB Rehearsal Dry-run Orchestrator Helper

Added a pure orchestrator helper that composes approval contract, rehearsal guard, plan builder, and report shape helper into one deterministic dry-run result shape.

### Step 7: Future Copied DB Write Rehearsal Command Design

Documented a future command boundary. The command remains hypothetical. No CLI entrypoint, package script, DB copy, DB write, report JSON write, API/UI flow, or product import route was added.
