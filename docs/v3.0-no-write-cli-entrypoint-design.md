# v3.0 No-write CLI Entrypoint Design

## Purpose

This document defines the proposed location, connection flow, safety boundaries, and exit behavior for a future HRC import preview CLI entrypoint before any actual entrypoint implementation exists.

The future entrypoint is intended to stay no-write by default. It must not connect product import routes, write to the database, generate report files, or become an automatic import path.

## Current Status

- No-write CLI command skeleton exists.
- Command output renderer exists.
- Dry-run orchestration helper exists.
- Copied DB path guard exists.
- Product import route is not connected.
- `package.json` script is not added.
- Actual CLI entrypoint is not implemented.

## Proposed Entrypoint Location

Candidate locations:

- `apps/server/src/cli/importHrcPreview.ts`
- `tools/import-hrc-preview.ts`

Recommendation:

- Do not connect the first entrypoint implementation to a `package.json` script.
- Implement the actual entrypoint only in a separate step after this design is accepted.
- Keep the entrypoint outside server runtime request handling.
- Keep the entrypoint disconnected from product import routes.

## Future Entrypoint Responsibilities

A future entrypoint may be responsible for:

- Reading `process.argv`.
- Passing caller arguments into the core helper.
- Building the no-write CLI command plan.
- Building the dry-run orchestration result.
- Building the command report.
- Rendering safe output text through the command output renderer.
- Printing rendered text to the console.
- Setting `process.exitCode`.

Important limits:

- This step does not implement any of those responsibilities.
- This step documents the future flow only.

## Entrypoint Non-responsibilities

Even a future no-write entrypoint must not perform:

- DB write.
- DB schema migration.
- Product import route activation.
- API call.
- UI interaction.
- Raw HRC zip read.
- External artifact JSON dependency.
- Report JSON file write.
- Backup file creation.
- Restore rehearsal.
- `package.json` script auto-wiring.
- `--write` flag support.
- `--force` flag support.

## Future Data Flow

Candidate no-write CLI flow:

1. `process.argv`
2. `parseHrcImportNoWriteCliArgs`
3. `guardHrcCopiedDbTargetPath`, if copied DB path is supplied
4. `buildHrcImportDryRunOrchestration`
5. `buildHrcImportCommandReport`
6. `renderHrcImportCommandReportText`
7. Console output
8. `process.exitCode`

This step documents the flow only. It does not wire `process.argv`, console output, or process exit behavior.

## Safety Policy

The future entrypoint must enforce these conditions:

- Default dry-run only.
- `writeAllowed` remains `false`.
- `dbWriteAllowed` remains `false`.
- `reportFileWriteAllowed` remains `false` unless separately approved in a later version.
- `--write` is rejected.
- `--force` is rejected.
- `--db-write` is rejected.
- `--production-db` is rejected.
- `--raw-hrc-path` is rejected.
- `--output-json` is rejected until separately planned.
- Production DB path is rejected.
- Local/private/raw path exposure is rejected.
- Raw HRC zip is not read.
- Product import route remains disabled.

## Exit Behavior

Candidate future exit behavior:

- `exitCode 0`: dry-run completed with no blocking issue.
- `exitCode 1`: validation blocking issue.
- `exitCode 2`: safety gate failure.
- `exitCode 3`: privacy/path failure.
- `exitCode 4`: internal/input shape error.

Manual review candidates may be non-fatal. Duplicate, missing-key, privacy, and safety issues are fatal.

## Implementation Guardrails For Future Step

An actual no-write CLI entrypoint implementation step must verify:

- Adding a `package.json` script requires separate approval.
- Entrypoint remains no-write only.
- Entrypoint performs no DB read/write.
- Entrypoint generates no report JSON.
- Entrypoint connects no product import route.
- Entrypoint creates no generated artifact.
- Entrypoint outputs no forbidden strings.
- Tests cover allowed dry-run args.
- Tests cover rejected write and force flags.
- Tests cover copied DB path guard behavior.
- Tests cover output renderer behavior.
- Tests cover exit code mapping.
- Tests cover no DB, filesystem, or API usage.

## Non-goals

This step does not include:

- Actual entrypoint file creation.
- `package.json` script.
- `process.argv` wiring.
- `console.log` implementation.
- `process.exitCode` implementation.
- DB read/write.
- File write.
- Report JSON generation.
- API/UI/runtime changes.
- Product import route.
- Raw HRC artifact handling.

## Future Step Candidates

- No-write CLI entrypoint implementation without `package.json` script.
- No-write CLI entrypoint tests.
- `package.json` script design, with separate approval required.
- Copied DB rehearsal manifest helper.
- Report JSON generation remains forbidden until separately planned.
