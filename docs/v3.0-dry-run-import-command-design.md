# v3.0 Dry-run-only Import Command Design

## 1. Purpose

Before HRC product import can ever be connected to a real DB write path, v3.0 defines the command-layer dry-run-only policy.

A future v3.0 command must not be write-capable by default. This Step 7 is docs-only and does not implement a command, add a package script, read or write the DB, write files, connect product import routes, or change API/UI/runtime behavior.

## 2. Current Status

- Import preview contract exists.
- Import preview validator exists.
- Existing solution snapshot adapter exists.
- Backup/rollback design exists.
- Backup manifest pure helper exists.
- Product import route is not connected.
- DB write is not allowed.
- DB read is not allowed in this design step.

## 3. Proposed Future Command Name

Candidate command names:

- `npm run import:hrc:preview`
- `npm run import:hrc:dry-run`

Policy:

- Do not add a `package.json` script in this Step 7.
- Do not implement the command in this Step 7.

## 4. Command Mode Policy

Future command policy:

- Default mode: dry-run only.
- Write mode: not available in v3.0 Step 7.
- `--write` flag: not implemented.
- `--force` flag: not implemented.
- Cannot be executed by GET/API/UI.
- Produces import preview only.
- Does not change the DB.
- Does not enable the product import route.

## 5. Input Policy

Allowed future input candidates:

- Sanitized in-memory preview rows.
- v2.9 classification summary.
- Existing canonical key snapshot.
- Backup manifest preview input.

Forbidden inputs:

- Raw HRC zip direct read.
- Raw HRC source folder direct read.
- External artifact JSON direct dependency.
- Product DB direct write.
- Uncontrolled filesystem scanning.
- Public-facing local path exposure.

## 6. Output Policy

Future command output policy:

- Console summary is allowed.
- Sanitized report output is allowed only if explicitly planned in a later step.
- Generated report JSON is not created in this Step 7.
- Local absolute path output is forbidden.
- Raw HRC path output is forbidden.
- Private user identifier output is forbidden.
- `dbWriteAllowed` remains always `false`.
- `writeAllowed` remains always `false`.

## 7. Exit Code Policy

Candidate future exit codes:

- `0`: dry-run preview completed with no blocking safety issue.
- `1`: blocking validation issue.
- `2`: safety gate failure.
- `3`: privacy/path scan failure.
- `4`: unexpected internal error.

Manual-review candidates do not necessarily fail the command. Privacy failure, DB write attempt, or raw path exposure must fail.

## 8. Safety Gates

Future dry-run command gates:

- DB SHA256 before command recorded.
- DB SHA256 after command confirmed unchanged.
- `git status` clean.
- Raw zip absence.
- Generated artifact absence.
- `artifacts/hrc-dry-run-reports` absence.
- Product import route disabled.
- `dbWriteAllowed` true count `0`.
- `writeAllowed` false.
- Privacy/path safe output.
- Duplicate/canonical key preview summary.

## 9. Integration With Previous v3.0 Steps

This Step 7 design may later compose these helpers:

- `hrcImportPreviewContract`
- `hrcImportPreviewValidator`
- `hrcExistingSolutionSnapshot`
- `hrcImportBackupManifest`

This Step 7 does not implement that orchestration command.

## 10. Non-goals

This Step 7 does not include:

- Command implementation.
- `package.json` script.
- DB read.
- DB write.
- File write.
- Actual report JSON generation.
- Actual backup.
- Restore rehearsal.
- API endpoint.
- UI button.
- Product import route.
- Schema migration.
- Raw HRC artifact handling.
