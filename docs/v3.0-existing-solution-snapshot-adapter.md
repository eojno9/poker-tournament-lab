# v3.0 Existing Solution Snapshot Adapter

## Purpose

The existing solution snapshot adapter builds an in-memory canonical key snapshot from existing product solution summary rows supplied by a caller.

Its purpose is to provide safe `existingCanonicalKeys` input for the v3.0 Step 3 import preview validator without reading the product DB directly.

## Non-Goals

- Not a DB read implementation.
- Not a DB write implementation.
- Not a DB schema migration.
- Not a product import route connection.
- Not an API change.
- Not a UI change.
- Not a server runtime adapter.
- Not a raw HRC zip reader.
- Not an external artifact JSON reader.

## Inputs

The adapter accepts in-memory rows with a deliberately small shape:

- `id`
- `canonicalKey`
- `source`
- `sourceFile`
- optional lightweight context fields such as `treeConfigId`, `heroPosition`, `tableSize`, and `remainingPlayers`

The adapter is not coupled to a concrete DB row type. A later server-layer source adapter may provide these rows, but that is outside this step.

## Outputs

The adapter returns an `HrcExistingSolutionCanonicalKeySnapshot`:

- `totalRows`
- `canonicalKeys`
- `entries`
- `missingCanonicalKeyCount`
- `duplicateCanonicalKeyCount`
- `uniqueCanonicalKeyCount`
- `sourceBreakdown`
- `warnings`

`canonicalKeys` is the unique, first-seen ordered canonical key list intended for `validateHrcImportPreviewRows`.

## Duplicate Policy

- `canonicalKey` values are trimmed.
- Case is preserved.
- Duplicate detection uses the trimmed canonical key.
- Duplicate rows do not throw.
- Duplicate rows are marked with `isDuplicate: true`.
- `duplicateCanonicalKeyCount` counts duplicate occurrences beyond the first occurrence.
- `canonicalKeys` remains deduped.

## Missing Canonical Key Policy

Rows with `null`, `undefined`, or empty canonical keys are counted as missing.

Missing rows remain visible in `entries` with `normalizedCanonicalKey: null` and a warning.

## Source Breakdown Policy

`sourceBreakdown` counts rows by sanitized source label.

Path-like or private source labels are redacted. `sourceFile` is reduced to a filename-like value so this adapter does not emit raw local paths.

## Step 3 Validator Connection

The intended preview-only flow is:

1. A caller supplies existing solution summary rows in memory.
2. `buildHrcExistingSolutionCanonicalKeySnapshot(rows)` creates the snapshot.
3. `snapshot.canonicalKeys` is passed to `validateHrcImportPreviewRows`.
4. The validator reports `DUPLICATE_EXISTING_DB` for preview rows whose canonical keys already exist.

No DB read or write occurs in this flow.
