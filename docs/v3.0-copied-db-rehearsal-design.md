# v3.0 Copied-DB-only Rehearsal Design

## 1. Purpose

This document fixes the policy for any future import write rehearsal: it must run only against a copied DB created outside the repo, never against the production product DB. The goal is to make rehearsal safety explicit before any write-capable import work is considered.

This step is docs-only. It does not copy the DB, create a copied DB, run a rehearsal, read or write the DB, implement a command, add a package script, call an API, change UI, or connect product import routes.

## 3. Core Policy

Future copied-DB rehearsal policy:

- Direct write rehearsal on the production DB is forbidden.
- Direct production DB mutation is forbidden.
- Write rehearsal is allowed only on a copied DB after explicit approval.
- The copied DB must live outside the repo.
- Public-facing docs and reports must use placeholders instead of local absolute copied DB paths.
- Copied DB rehearsal is forbidden without explicit user approval.
- The production DB SHA256 must remain unchanged before and after copied DB rehearsal.
- Copied DB rehearsal must not connect to product import routes.
- Copied DB rehearsal must not connect to UI buttons or runtime app flows.

## 4. Future Copied DB Location Policy

Expected placeholder:

`<local-backup-root>/v3.0-copied-db-rehearsal-YYYYMMDD-HHMMSS/poker-tournament-lab.db`

Rules:

- Do not expose actual local paths in repo docs or GitHub-safe materials.
- Do not place copied DB files inside the repo.
- Do not include raw HRC zip files in copied DB rehearsal folders.
- Do not create generated dry-run artifacts unless a later explicit step approves them.
- Do not commit copied DB files, copied DB folders, raw zip files, or generated rehearsal reports.

## 5. Pre-rehearsal Gates

A future copied DB rehearsal must pass these gates before execution:

- Explicit user approval.
- Git status clean.
- `npm.cmd run test` PASS.
- `npm.cmd run build` PASS.
- `npm.cmd run test:smoke` PASS.
- Production DB SHA256 before value recorded.
- Copied DB SHA256 before value recorded.
- Backup manifest preview prepared.
- Import preview validation PASS.
- Duplicate/canonical key preview PASS.
- Privacy/path scan PASS.
- Raw zip absence confirmed.
- Generated artifact JSON absence confirmed.
- `artifacts/hrc-dry-run-reports` absence confirmed unless a later step explicitly approves it.
- Product import route disabled.
- UI/API not connected to write-capable import flow.

## 6. Rehearsal Execution Policy

If copied DB rehearsal is implemented later:

- The command must require a copied DB path.
- The command must refuse the production DB path.
- The command must avoid exposing repo-local or user-local backup paths in public reports.
- The command must run dry-run by default.
- Write rehearsal mode must require an explicit copied-DB-only flag.
- Write rehearsal mode must never target the production DB.
- The command must record an import batch id.
- The command must record a backup or rehearsal id.
- The command must record source provenance.
- The command must record copied DB SHA256 before and after.
- The command must verify that production DB SHA256 is unchanged.

## 7. Post-rehearsal Verification Policy

After a future copied DB rehearsal:

- Production DB SHA256 unchanged.
- Copied DB SHA256 changed only if write rehearsal was explicitly approved.
- `npm.cmd run test` PASS.
- `npm.cmd run build` PASS.
- `npm.cmd run test:smoke` PASS.
- Existing exact/random lookup regression absent.
- Duplicate canonical key remains `0`.
- Near-match HRC false positives remain `0`.
- `forbiddenHrcHit` true count remains `0`.
- Rollback instructions are present and reviewed.
- Rehearsal report is privacy/path redacted.

## 8. Abort Policy

Abort immediately if any of these conditions occur:

- Production DB path is selected as the target.
- Product import route is enabled.
- UI/API can trigger import execution.
- Production DB SHA256 changes.
- Raw local path exposure appears in output.
- Privacy/path scan fails.
- Raw HRC zip is detected in the repo or rehearsal output.
- Generated artifact or rehearsal report appears unexpectedly.
- Duplicate canonical key is detected.
- Rollback plan is missing.
