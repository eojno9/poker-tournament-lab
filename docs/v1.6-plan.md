# v1.6 Plan - DB Action/Sizing Selector Foundation

## 1) Summary
v1.6은 DB에 실제로 존재하는 action/raise size만 안전하게 추출, 표시, 선택 보조할 수 있도록 기반을 만드는 버전입니다.

## 5) Workstream Results

### A. Canonical Action/Size Audit
- `buildActionSizingAudit` was added as a read-only helper.
- It checks actionPath, treeConfig, source metadata, strategy action, explicit size-like fields, and canonical sensitivity.
- Canonical key generation was not changed.
- Finding: actionPath differences affect canonical key. treeConfig is currently outside canonical key input, so size tokens found only in treeConfig need warning/visibility.

### B. Available DB Action/Sizing Extractor
- `extractAvailableActionSizingOptions` was added as a pure read-only extractor.
- It returns only action/sizing candidates found in existing solution data.
- It does not invent missing raise sizes.
- Explicit numeric size fields are high confidence.
- actionPath/treeConfig/source metadata parsing is medium/low confidence depending on source.
- Missing size signals are separated as `UNKNOWN/UNSPECIFIED` with warnings.

### C. Analyze DB-Driven Selector
- Analyze screen now shows `DB 기준 액션/사이즈 후보`.
- It reuses existing `/api/solutions`; no new API was added.
- Candidate selection fills form fields only.
- Candidate selection does not auto-run Analyze.
- It explicitly warns that DB-missing sizes are not treated as `HRC_PRECOMPUTED_DB` exact matches.

### D. Database Detail Action/Sizing Display
- Database detail now shows `Action / Sizing Summary`.
- It displays actionPath, treeConfig, detected actions, detected raise sizes, all-in/shove, sizeKind, sizeLabel, confidence, sourceCount, size signals, and explicit size fields.
- It uses selected solution data only.
- It does not create or recommend sizes absent from DB data.
- `UNKNOWN/UNSPECIFIED` means imported data lacks explicit size information.
