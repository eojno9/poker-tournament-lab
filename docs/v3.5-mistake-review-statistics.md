# v3.5 mistake review and statistics design

## Scope

This document defines the planned mistake review and local statistics design for
v3.5 Trainer UX.

This is a docs-only design step.

No implementation, dependency change, DB access, raw data access, generated
artifact creation, API integration, UI implementation, GitHub push, or tag
change is performed in this step.

## Safety Constraints

- Off-table study only
- No RTA
- No live assistance
- No OCR
- No screen capture
- No overlay
- No hotkey
- No live watcher
- No poker client integration
- No production DB write
- No raw DB/raw zip access
- No bundled data artifacts
- Local-only Trainer persistence

## Design Goals

| Goal | Description |
| --- | --- |
| Clear mistake queue | Let users revisit incorrect answers from local Trainer history |
| Retry flow | Allow off-table retry without implying live-game advice |
| Resolved status | Track whether a mistake has been corrected locally |
| Local statistics | Show accuracy and mistake patterns from local attempts only |
| Safe reset | Allow clearing local mistakes/history without server involvement |
| Beginner-friendly feedback | Explain mistakes without overclaiming precision |

## Mistake Item Model

Candidate shape:

```ts
type TrainerMistakeItem = {
  version: 1;
  mistakeId: string;
  firstAttemptId: string;
  latestAttemptId?: string;
  solutionId: string;
  hand: string;
  position?: string;
  tableSize?: string;
  treeType?: string;
  source: "LOCAL_PRECOMPUTED_STUDY_DATA";
  selectedAction: string;
  correctAction?: string;
  firstWrongAt: string;
  lastReviewedAt?: string;
  retryCount: number;
  status: "unresolved" | "resolved" | "dismissed";
};
```

Design notes:

- This is local-only off-table study metadata.
- Do not include live hand import metadata.
- Do not include OCR, screen capture, poker client, table watcher, or hotkey
  metadata.
- Do not include private raw data paths, specific data source names, providers,
  permission details, or original file names.

## Mistake Queue Rules

| Event | Rule |
| --- | --- |
| Incorrect answer | Add or update unresolved mistake |
| Correct retry | Mark mistake as resolved |
| Incorrect retry | Increment retry count and keep unresolved |
| User dismisses mistake | Mark dismissed, do not delete immediately unless reset |
| Clear mistakes | Remove local mistake queue only |
| Corrupt localStorage | Reset safely and show non-alarming message |

## Retry Flow

Planned flow:

1. User opens Mistake Review
2. App loads local mistake queue
3. User selects unresolved mistake
4. Trainer shows same off-table study spot
5. User answers again
6. Feedback is shown
7. Correct retry marks resolved
8. Incorrect retry remains unresolved
9. Summary updates local stats

No server storage, live game detection, OCR, screen capture, or poker client
integration is allowed.

## Mistake Review States

| State | UI intent |
| --- | --- |
| No mistakes | Explain that mistakes appear after incorrect Trainer answers |
| Unresolved mistakes | Show retry queue |
| Resolved mistakes | Show completed review items, optionally collapsible |
| Dismissed mistakes | Hide by default, clearable |
| Corrupt local data | Reset safely |
| Unsupported item | Explain this study spot is not available for retry |

## Local Statistics Model

Candidate shape:

```ts
type TrainerLocalStats = {
  version: 1;
  totalAttempts: number;
  correctAttempts: number;
  incorrectAttempts: number;
  skippedQuestions: number;
  accuracyPct: number;
  recentAccuracyPct?: number;
  byPosition?: Record<string, TrainerStatBucket>;
  byAction?: Record<string, TrainerStatBucket>;
  bySource?: Record<string, TrainerStatBucket>;
  unresolvedMistakes: number;
  resolvedMistakes: number;
};
```

Candidate bucket:

```ts
type TrainerStatBucket = {
  attempts: number;
  correct: number;
  incorrect: number;
  accuracyPct: number;
};
```

## Statistic Calculation Rules

| Stat | Rule |
| --- | --- |
| totalAttempts | Count local submitted attempts |
| correctAttempts | Count local attempts with `isCorrect = true` |
| incorrectAttempts | Count local attempts with `isCorrect = false` |
| skippedQuestions | Count skipped items only if skip is implemented |
| accuracyPct | `correctAttempts / totalAttempts * 100`, safe rounded |
| recentAccuracyPct | Same formula over recent N attempts |
| byPosition | Group local attempts by position |
| byAction | Group local attempts by selected action |
| unresolvedMistakes | Count local mistakes with `status = unresolved` |
| resolvedMistakes | Count local mistakes with `status = resolved` |

Rules:

- If denominator is 0, show no percentage or 0 with clear empty-state copy.
- Label all statistics as local Trainer history only.
- Do not imply solver-grade precision beyond available local attempts.
- Treat derived caches as disposable; local attempts and mistake records are the
  source of truth.

## localStorage Plan

Candidate keys:

| Key | Purpose | Version |
| --- | --- | --- |
| `ptl.trainer.mistakes.v1` | Local mistake queue | v1 |
| `ptl.trainer.recentAttempts.v1` | Recent local attempts | v1 |
| `ptl.trainer.statsCache.v1` | Optional derived stats cache | v1 |

Rules:

- Prefer deriving stats from recent attempts/mistakes.
- Stats cache is optional and must be safely discardable.
- No server persistence.
- No account tracking.
- Safe parse fallback required.
- Clear/reset action required.

## UX Copy Requirements

Recommended public-safe copy:

- `Mistakes are saved locally on this device.`
- `This Trainer is for off-table study only.`
- `No live assistance or poker client integration is provided.`
- `Accuracy is based only on your local Trainer attempts.`
- `No matching mistakes yet.`

Avoid:

- real-time recommendation wording
- live play decision wording
- monitoring/importing live hands
- guaranteed improvement claims
- raw data/source/provider disclosure

## Future Implementation Guardrails

Later implementation must include tests for:

- incorrect answer adds unresolved mistake
- correct retry marks resolved
- incorrect retry increments retry count
- clear mistakes removes local mistake queue
- corrupt localStorage falls back safely
- accuracy denominator 0 handled safely
- by-position and by-action stats calculated correctly
- no server/API persistence introduced
