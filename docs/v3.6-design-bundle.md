## Session-Based Study Mode Design

### Goals

Session mode should make off-table practice feel more intentional without
creating live-play behavior.

Candidate goals:

- configure a local practice session before starting
- show progress during the session
- keep answer/feedback flow stable
- show a clear completion summary
- allow reset without corrupting local history
- preserve current local-only Trainer state

### Session Model

Candidate fields:

| Field | Purpose |
| --- | --- |
| `sessionId` | local session identifier |
| `status` | not started, in progress, completed, reset |
| `filters` | selected local study filters |
| `questionLimit` | optional local question target |
| `currentIndex` | progress in the local session |
| `attemptIds` | local attempt references |
| `startedAt` | local timestamp |
| `completedAt` | local timestamp, optional |
| `summary` | derived local session summary |

### UX Flow

Candidate flow:

1. User reviews local filters and starts a session
2. Trainer shows progress and current study spot
3. User answers
4. Feedback is shown
5. Attempt is stored locally
6. Mistake queue updates locally if needed
7. User continues until completion or reset
8. Completion summary shows local stats for the session

### Boundaries

- no server session persistence
- no account or cloud storage
- no live hand detection
- no poker client integration
- no hidden import or DB read path

## Position And Action Local Statistics Design

### Goals

Statistics should help the user understand local practice patterns without
overstating precision.

Candidate goals:

- show by-position local accuracy
- show by-action local accuracy
- show unresolved mistakes by position/action where available
- keep denominator-0 states safe
- keep labels compact and Korean-first
- avoid dense analytics before there is enough local history

### Candidate Buckets

| Bucket | Data source | Display rule |
| --- | --- | --- |
| Position | local Trainer attempts | show only if attempts exist |
| Action | local Trainer attempts | show only if attempts exist |
| Recent attempts | local Trainer attempts | compact trend only |
| Mistake status | local mistake queue | unresolved/resolved counts |

### Calculation Rules

- derive statistics from local attempts and local mistake records
- do not store derived stats unless the cache is discardable
- if attempts are 0, show an empty state rather than a misleading percentage
- round percentages consistently
- label results as local Trainer history only

## localStorage Migration And Fallback Design

### Goals

v3.6 should prepare safer localStorage handling without changing the storage
boundary.

Candidate goals:

- keep versioned keys
- parse safely
- recover from corrupt local JSON
- avoid data loss surprises
- provide local reset behavior
- make future migration steps explicit

### Key Inventory

Expected current keys:

- `ptl.trainer.recentAttempts.v1`
- `ptl.trainer.mistakes.v1`

Potential future keys:

- `ptl.trainer.sessions.v1`
- `ptl.trainer.settings.v1`

### Fallback Rules

| Condition | Expected behavior |
| --- | --- |
| Missing key | use empty default |
| Invalid JSON | reset safely and show calm local fallback copy |
| Wrong version | ignore or migrate in a dedicated step |
| Quota failure | keep UI usable and show local-save warning |
| Clear local data | remove only Trainer local keys |

### Migration Rules

- do not silently mutate unknown data shapes
- keep migration helpers pure where practical
- test corrupt and missing-key cases
- do not introduce network writes
- do not include raw path/source/provider/permission metadata

## Korean-First Copy Audit Checklist

Implementation should include a copy audit for touched Trainer surfaces:

| Area | Check |
| --- | --- |
| Trainer top guidance | concise Korean-first copy |
| Filter/options labels | Korean-first and understandable |
| Question card | prompt and context are clear |
| Answer feedback | correct/incorrect copy is calm and direct |
| Mistake review | queue states and retry action are Korean-first |
| Session mode | start/progress/complete/reset copy is Korean-first |
| Statistics | local-only scope is clear |
| Empty states | actionable and non-alarming |
| Safety copy | off-table/no-live-assistance boundary visible |

Copy audit rules:

- keep technical terms only where precision matters
- avoid live-use wording
- avoid data source/provider/permission disclosure
- avoid overclaiming accuracy or improvement
- keep the UI practical rather than instructional-heavy
