# v3.1 Copied DB Rehearsal Report Shape Helper

## Purpose

The copied DB rehearsal report shape helper turns a copied DB rehearsal plan into a deterministic in-memory report and summary shape. It exists to describe what a future rehearsal report should contain without writing a report file or running any import behavior.

The helper accepts Step 4 plan output plus counts, SHA summary, safety summary, and execution summary inputs.

## Non-goals

- No actual DB copy.
- No actual DB write.
- No production DB write.
- No DB schema migration.
- No product import route connection.
- No API/UI import flow.
- No `package.json` script.
- No raw HRC zip access, copy, extraction, or commit.
- No report JSON file generation.
- No filesystem access.
- No DB access.
- No `process.env`, `Date.now`, or random value usage.

## Relationship to the Plan Builder

Step 4 produces a plan status:

- `READY_FOR_DRY_RUN`
- `READY_FOR_COPIED_DB_WRITE_REHEARSAL`
- `BLOCKED`

Step 5 wraps that plan into a report shape with:

- decision flags
- counts
- DB SHA evidence
- safety evidence
- execution evidence
- blocked reasons
- warnings
- required next checks
- exit code

The report helper does not execute any plan step.

## Report Status and Decision

`READY_FOR_DRY_RUN` means the report shape describes a dry-run-only path. It can be summarized in memory but cannot write files.

`READY_FOR_COPIED_DB_WRITE_REHEARSAL` means the plan inputs say a future copied DB rehearsal could be considered. The report still performs no write.

`BLOCKED` means the plan, safety evidence, execution evidence, or SHA evidence blocks the rehearsal.

Decision flags:

- `canDryRun`
- `canCopiedDbWriteRehearsal`
- `canProductionDbWrite`
- `canWriteReportFile`

`canProductionDbWrite` is always `false`.

`canWriteReportFile` is always `false` in this step.

## Exit Code Policy

- `0`: ready report shape with no blocking issue
- `1`: validation or plan blocked
- `2`: safety blocked
- `3`: privacy/path blocked
- `4`: internal/input shape blocked

Privacy failure takes precedence over general safety failures. Safety failures include raw zip presence, artifact report presence, product route connection, API/UI runtime changes, actual DB copy, actual DB write, report JSON write, and original DB SHA mismatch.

## Safety Policy

The helper blocks or warns on evidence that should not exist in this step:

- `actualDbCopyPerformed: true` blocks the report as a safety failure.
- `actualDbWritePerformed: true` blocks the report as a safety failure.
- `reportJsonWritten: true` blocks the report as a safety failure.
- `privacyScanPassed: false` blocks with exit code `3`.
- `rawZipAbsent: false` blocks with exit code `2`.
- `productRouteDisconnected: false` blocks with exit code `2`.
- original DB SHA mismatch blocks with exit code `2`.
- `importPreviewAllowed: 0` creates a warning.

## Privacy and Path Redaction

The report output must not expose real local paths, private user tokens, emails, or raw HRC paths. Any private strings in IDs, SHA fields, warnings, blocked reasons, or next checks are redacted.
