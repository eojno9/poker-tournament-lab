# v3.0 No-write CLI Command Skeleton

## Purpose

This step defines a future HRC import preview CLI skeleton as a core-level pure helper. It parses caller-supplied argument arrays into a no-write command plan that later tooling can use, while keeping every write, DB, filesystem, API, UI, and runtime connection disabled.

The helper is not an actual CLI entrypoint. It does not read process arguments directly, print to the console, add a package script, read or write files, access the DB, generate report JSON, or connect product import routes.

## Non-Goals

- No actual CLI entrypoint.
- No package script.
- No direct process argument connection.
- No console output.
- No DB read or write.
- No filesystem access.
- No report JSON generation.
- No API, UI, server runtime, or product import route connection.
- No DB copy, copied DB creation, or rehearsal execution.
- No raw HRC zip or external artifact access.

## Allowed Command Names

- `import:hrc:preview`
- `import:hrc:dry-run`

The command plan mode is always `DRY_RUN`.

## Allowed Flags

- `--dry-run`
- `--help`
- `--copied-db-path <path>`

`--copied-db-path` is preview-only in this step. It does not read, copy, create, or open a DB file.

## Forbidden Flags

These flags are always rejected:

- `--write`
- `--force`
- `--db-write`
- `--production-db`
- `--raw-hrc-path`
- `--output-json`

The skeleton records rejected flags in the command plan and keeps all write allowances false.

## Copied DB Path Guard Integration

When `--copied-db-path <path>` is provided, the helper calls the copied DB path guard as a pure string validation step. The guard can allow a copied DB under the approved backup-root placeholder policy, or block production DB targets, repo-local targets, raw HRC paths, invalid extensions, and SQLite sidecar files.

The helper does not check whether the path exists. Output contains only redacted path information.

## Write Policy

These command plan fields are fixed:

- `dryRunOnly: true`
- `writeAllowed: false`
- `forceAllowed: false`
- `dbWriteAllowed: false`
- `reportFileWriteAllowed: false`

Any future write-capable flow must be separate, copied-DB-only, explicitly approved, and still forbidden from touching the production DB.

## Privacy And Output Policy

Command plan output must not expose:

- local absolute paths
- user tokens
- private relay or email-like values
- raw HRC path markers
- raw HRC zip paths

The helper returns data only. It does not print.

## Exit Code Preview Policy

- `0`: accepted no-write plan or help output.
- `1`: forbidden write/force/output flag or unsupported flag.
- `2`: copied DB path guard safety block.
- `3`: raw/private path failure.
- `4`: invalid command shape or missing required flag value.
