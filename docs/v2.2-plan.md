## Data Principles

### Real HRC / Export Data vs Test Fixture

- 실제 HRC/export 기반 데이터와 테스트 fixture 데이터를 명확히 분리합니다.
- 테스트 fixture는 UI, 분류, 필터, validator coverage 검증용입니다.
- 테스트 fixture를 실제 전략 추천, solved equilibrium, HRC exact result처럼 표현하지 않습니다.

### TEST_ONLY / SAMPLE Metadata

sample fixture에는 source/metadata에 SAMPLE 또는 TEST_ONLY 성격을 명확히 남깁니다.

권장 metadata:

- `sourceLabel`: `SAMPLE_TEST_ONLY` 또는 유사 라벨
- `sourceMetadata.isSample`: `true`
- `sourceMetadata.testOnly`: `true`
- `sourceMetadata.calculationModel`: `sample-fixture-not-solver`
- `sourceMetadata.exportShape`: `multi-action-v2-sample`
- `sourceMetadata.streetScope`: `preflop-only`
- `sourceMetadata.actionTags`: node별 action tree tags

### No Recommendation Semantics

- 없는 spot을 자동 생성하지 않습니다.
- nearest recommendation을 하지 않습니다.
- solver 계산 결과를 임의로 만들어 실제 전략처럼 취급하지 않습니다.
- EV/frequency 값이 fixture에 있더라도 테스트용 sample value로 표시하고 실제 GTO 추천처럼 문서화하지 않습니다.

## Sample Spot Candidates

### RFI / Open Raise

- 예: BTN first-in
- action node: `RFI` / `OPEN_RAISE`
- candidate actions: `RAISE 2.2bb`, `FOLD`, `ALL_IN`
- 목적: first-in open raise node, raise size label, mixed action matrix 표시 검증

### Limp

- 예: SB first-in
- action node: `OPEN_LIMP`
- candidate actions: `LIMP`, `RAISE`, `FOLD`, `ALL_IN`
- 목적: LIMP와 CALL 분리, LIMP badge/metadata/availableActions 표시 검증

### Facing Open

- 예: BB vs BTN open
- action node: `VS_OPEN`
- candidate actions: `CALL`, `RAISE`, `FOLD`, `ALL_IN`
- 목적: CALL이 LIMP와 분리되는지, facing-open breadcrumb와 action filters 검증

### Facing Limp

- 예: BB vs SB limp
- action node: `VS_LIMP`
- candidate actions: 현재 action model에 맞춰 `CHECK`, `CALL`, `RAISE`, `FOLD` 중 명확한 조합 사용
- 목적: facing limp 분류, available action/size filter, empty/warning state 검증

### 3bet / vs 3bet

- 예: BTN open 후 blind 3bet 또는 opener vs 3bet
- action node: `THREE_BET` 또는 `VS_THREE_BET`
- candidate actions: `RAISE`, `CALL`, `FOLD`, `ALL_IN`
- 목적: 3bet token을 단순 RAISE와 혼동하지 않는지 검증

## Sample Strategy v2 actions[] Shape

sample solution strategy는 v1.8 multi-action v2 shape를 따릅니다.

각 hand는 `actions[]`를 가질 수 있습니다.

각 action은 다음 필드를 사용할 수 있습니다.

- `action`: `FOLD` / `CHECK` / `CALL` / `BET` / `RAISE` / `ALL_IN` / `UNKNOWN` 및 LIMP 표시가 필요한 경우의 확장 방향
- `sizeBb`
- `sizePctPot`
- `isAllIn`
- `rawSizeLabel`
- `frequency`
- `ev`
- `chipEv`
- `icmEv`
- `sourceActionLabel`
- `warnings`

처리 원칙:

- 없는 값을 0으로 채우지 않습니다.
- missing EV는 `제공되지 않음`으로 표시 가능한 상태를 유지합니다.
- size 없는 RAISE/BET/CALL은 warning으로 드러냅니다.
- ALL_IN은 explicit raise size가 없어도 valid하게 처리합니다.
- UNKNOWN action은 warning으로 드러냅니다.

- `calculationModel`: `sample-fixture-not-solver`
- `spotFamily`: `RFI`, `LIMP`, `FACING_OPEN`, `FACING_LIMP`, `THREE_BET`, `VS_THREE_BET`
- `exportShape`: `multi-action-v2-sample`
- `streetScope`: `preflop-only`
- `actionTags`: 예: `RFI`, `OPEN_RAISE`, `LIMP`, `VS_OPEN`, `VS_LIMP`, `THREE_BET`, `VS_THREE_BET`
- `isSample`: `true`
- `testOnly`: `true`
