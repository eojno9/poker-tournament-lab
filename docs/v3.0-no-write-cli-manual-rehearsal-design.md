# v3.0 No-write CLI Manual Invocation Rehearsal Design

## 1. Purpose

This document defines the safety conditions and verification procedure for a future manual rehearsal of the no-write CLI entrypoint implemented in Step 15.

This step is documentation only. It does not execute the entrypoint, wire a command, add a package script, read the DB, write files, generate reports, or connect runtime routes.

## 2. Current Status

- No-write entrypoint function exists.
- `package.json` script is not added.
- `process.argv` is not directly connected.
- `process.exit` is not used.
- `console.log` is not directly connected.
- DB read/write is not performed.
- Report JSON is not generated.
- Product import route is not connected.

## 3. Rehearsal Principle

A future manual rehearsal must follow these rules:

- Dry-run only.
- No DB read.
- No DB write.
- No file write.
- No report JSON generation.
- No product route.
- No UI/API trigger.
- No `package.json` script until separately approved.
- Input context must be in-memory and sanitized.
- Copied DB target path must pass the copied DB path guard.
- Production DB target must be rejected.
- Output must be sanitized.

## 4. Future Invocation Candidate

This step does not run any invocation.

Future candidates:

- Test harness invocation.
- Direct function invocation in a controlled dev-only script.
- Later optional CLI script only after separate approval.

Forbidden in this step and still blocked until separately approved:

- `package.json` script addition.
- `npm run import:hrc:preview` addition.
- Direct `process.argv` connection.
- Direct `console.log` connection.
- `process.exit`.
- Report JSON file output.

## 5. Required Input Context

A future manual invocation must inject the complete context explicitly:

- `previewRows`
- `existingSolutionRows`
- `classificationSummary`
- `safetyChecks`
- `dbSha256Before`
- `dbSha256After`
- `timestampIso`
- `branchName`
- `commitHash`
- `dbFileName`
- `targetDbPath`
- `productionDbPath`
- `repoRootPath`
- `backupRootPath`

Important limits:

- Do not read this context from the DB.
- Do not read this context from raw HRC artifacts.
- Do not read this context from external artifact JSON.
- Do not expose actual local paths in public-facing output.

## 6. Expected Result Policy

A future invocation result must verify:

- `exitCode`
- `stdoutLines`
- `stderrLines`
- report shape
- `writeAllowed: false`
- `dbWriteAllowed: false`
- `reportFileWriteAllowed: false`
- no forbidden string exposure
- DB SHA256 before/after unchanged
- product import route disabled
- raw zip absent
- generated artifact absent

## 7. Rehearsal Success Criteria

The future rehearsal succeeds only if:

- Valid dry-run fixture returns `exitCode 0`.
- v2.9 fixture reports `importPreviewAllowed 19`.
- `writeAllowed` is `false`.
- `dbWriteAllowed` is `false`.
- `reportFileWriteAllowed` is `false`.
- No DB read/write occurs.
- No file write occurs.
- No report JSON is generated.
- No API/UI/runtime connection is added.
- No forbidden string is output.
- DB SHA256 remains unchanged.
- `npm.cmd run test` passes.
- `npm.cmd run build` passes.
- `npm.cmd run test:smoke` passes.

## 8. Rehearsal Failure Criteria

The future rehearsal must fail or stop if any of these occur:

- Production DB target is accepted.
- Repo-local DB target is accepted.
- `--write` is accepted.
- `--force` is accepted.
- `--db-write` is accepted.
- `--production` is accepted.
- `--raw` or `--raw-hrc` is accepted.
- `--output-json` is accepted.
- `process.exit` is used.
- `console.log` is directly connected.
- DB SHA256 changes.
- Report JSON is generated.
- Forbidden string is exposed.

## 9. Non-goals

This step does not include:

- Actual manual invocation.
- `package.json` script.
- `process.argv` connection.
- `process.exit`.
- Direct `console.log` output.
- DB read/write.
- File write.
- Report JSON.
- API endpoint.
- UI button.
- Product import route.
- Schema migration.
- Raw HRC artifact handling.

- Optional package script design after no-write rehearsal PASS.
- Product import route remains disabled.
