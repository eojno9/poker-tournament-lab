## Goals

v3.7 should complete the remaining practical Trainer experience work before a
later v4.0 readiness push.

Primary goals:

- improve Trainer session start, progress, completion, and reset flow
- add local-only filter settings save/restore UX
- refine mistake review sorting, filtering, and state visibility
- improve local statistics time/range readability
- strengthen versioned localStorage migration and fallback rules
- run a Korean-first UI copy audit over touched Trainer surfaces
- perform baseline accessibility and keyboard navigation checks
- check narrow/mobile Trainer layout behavior
- turn the v4.0 public readiness checklist into a concrete implementation gate

## Non-Goals

v3.7 must not include:

- server-side Trainer persistence
- account, cloud, or cross-device history sync
- new dependencies
- package-lock changes without a separate approval step
- DB writes or production DB write paths
- raw data, raw archive, or generated report workflows
- API route expansion for Trainer state
- live/RTA behavior
- OCR
- screen capture
- overlay
- hotkey workflow
- live watcher
- poker client integration
- GitHub push or tag work in implementation steps

## Test And Smoke Plan

Implementation should run:

```powershell
npm.cmd run typecheck
npm.cmd run test
npm.cmd run build
npm.cmd run test:smoke
```

Focused test expectations:

- session start state appears before attempts
- session completion summary appears at the configured end condition
- session reset clears only local session UI state, not unrelated app state
- saved filter settings restore after reload/localStorage load
- corrupt filter settings fall back safely
- mistake review filters show correct unresolved/resolved/hidden counts
- mistake retry path still works
- local statistics denominator-0 states remain safe
- Korean-first labels remain visible in Trainer smoke
- safety copy remains visible and unchanged in meaning
