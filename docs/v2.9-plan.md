## v2.9 Goal

The goal is to perform a controlled real raw HRC batch rehearsal against the repo-external HRC zip folder, then use artifact reports, index/comparison summaries, and the dashboard to determine which zip files are plausible future import candidates.

The rehearsal should answer how many real raw HRC zip files can be read, recognized, converted to artifact-safe reports, reviewed in the dashboard, and classified without importing anything into the product database.

## Artifact Commit Policy

- Generated artifact JSON is not committed automatically.
- Artifacts created by the real batch rehearsal must pass privacy/path scanning and dashboard review before any commit is considered.
- Commit decisions for artifacts must be made in a separate explicit step.
- Release backups may include a reviewed artifact set or summary report only if the step explicitly requests it.
- Raw zip originals are never included in commits or backups.
- Raw zip full extraction output is never included in commits or backups.

## Batch Rehearsal Policy

- Batch rehearsal is opt-in and explicit.
- The raw input folder must be repo-external.
- Repo-internal zip inputs are rejected.
- Path traversal is rejected.
- Non-zip raw inputs are skipped or rejected according to the batch runner report policy.
- `--allow-repo-artifact-write` is required for artifact writes.
- `--write-index` is required for index/comparison output.
- `--continue-on-error` can be used so one malformed zip does not stop the full inventory.
- Failed zip files remain diagnostic reports only and are not product import payloads.
- Batch output must preserve:
  - `rawZipCommitted: false`
  - `productImportConnected: false`
  - `dbWriteApplied: false`
  - `apiUsed: false`
  - `uiUsed: false`

## Privacy And Path Redaction Policy

- Artifact JSON must not contain raw absolute paths.
- Artifact JSON must not contain local user tokens.
- Artifact JSON must not contain email addresses.
- Artifact JSON must not contain raw zip binary content.
- Artifact JSON must not contain full raw zip extraction content.
- Dashboard responses must expose masked/sanitized paths only.
- Mismatch samples remain capped and must not include long raw hand strategy dumps.
- Privacy warning artifacts are reviewed but not promoted as import candidates.

## Dashboard Review Policy

- Dashboard review is read-only.
- The dashboard may call only:
  - `GET /api/hrc-dry-run-artifacts`
  - `GET /api/hrc-dry-run-artifacts/:fileName`
- The dashboard must not call product import routes.
- The dashboard must not trigger the batch runner.
- The dashboard must not write to the DB.
- The dashboard must not read raw zip files.
- The dashboard must not display raw absolute paths, user tokens, or email addresses.
- Dangerous controls remain forbidden:
  - import
  - export
  - run
  - upload
  - delete
  - write
  - solver
  - analyze

## Import Candidate Classification Policy

The classification report should use these labels:

- `IMPORT_CANDIDATE`
- `HOLD`
- `EXCLUDE`
- `NEEDS_MANUAL_REVIEW`

Classification should record reasons such as:

- raw shape OK
- settings/node OK
- action/hand count OK
- adapter OK
- validator OK
- privacy OK
- mismatch warning
- missing node
- malformed JSON
- unsupported shape

Classification is advisory only. It does not import, write to the DB, or change product import logic.
