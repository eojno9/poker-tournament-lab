# v3.5 minimal Trainer UX implementation plan

## Scope

This document defines the minimal implementation plan for v3.5 Trainer UX
improvements.

This is a docs-only planning step.

No implementation, dependency change, DB access, raw data access, generated
artifact creation, API integration, UI implementation, GitHub push, or tag
change is performed in this step.

## Safety Constraints

- Off-table study only
- No RTA
- No live assistance
- No OCR
- No screen capture
- No overlay
- No hotkey
- No live watcher
- No poker client integration
- No production DB write
- No raw DB/raw zip access
- No bundled data artifacts
- Local-only Trainer persistence

## Implementation Goals

| Goal | Description | Priority |
| --- | --- | --- |
| Session flow clarity | Add clearer start-progress-feedback-summary Trainer flow | High |
| Mistake queue | Add local-only mistake review queue | High |
| Retry flow | Allow retrying incorrect local Trainer attempts | High |
| Local stats | Add local attempt and accuracy summary | Medium |
| Empty states | Add beginner-friendly empty and unsupported states | Medium |
| Safety copy | Keep off-table/no-live-assistance copy visible | High |

## Mistake Review Behavior Plan

| Event | Planned behavior |
| --- | --- |
| Incorrect answer | Add/update unresolved local mistake |
| Correct retry | Mark local mistake resolved |
| Incorrect retry | Increment retry count |
| Dismiss mistake | Mark dismissed locally |
| Clear mistakes | Clear local mistake queue only |
| Corrupt local data | Reset safely and show simple message |
