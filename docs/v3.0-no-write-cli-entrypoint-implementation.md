# v3.0 No-write CLI Entrypoint Implementation

## Purpose

This step implements the future HRC import preview entrypoint as a testable, no-write function without connecting it to an actual command runner.

The implementation lives at `apps/server/src/cli/importHrcPreview.ts`, but it is not registered as a server route, not wired into `package.json`, and not connected to product import behavior.

## Non-goals

- Not a `package.json` script.
- Not an auto-running CLI command.
- No direct `process.argv` connection.
- No process exit call.
- No direct console output.
- No DB read/write.
- No DB copy.
- No copied DB creation.
- No restore rehearsal.
- No file write.
- No report JSON creation.
- No API/UI/runtime connection.
- No product import route connection.
- No raw HRC zip access.
- No external artifact JSON access.

## Entrypoint Shape

The implementation exports pure functions:

- `parseHrcImportPreviewArgs(argv, context)`
- `runHrcImportPreviewEntrypoint(argv, context)`
- `renderHrcImportPreviewOutput(result)`

The entrypoint accepts only caller-supplied `argv` and an in-memory context. It does not read files, query the DB, inspect raw HRC zip files, write report JSON, or call product runtime routes.

## Context Policy

The context is injected by tests or a future no-write runner and includes:

- preview rows
- existing solution rows
- classification summary
- safety checks
- DB SHA256 before/after values
- timestamp, branch, and commit metadata
- DB file name
- target DB path policy inputs
- production DB, repo root, and backup root policy strings

All values are treated as untrusted display input and are redacted before they reach output lines or error lines.

## Argv Policy

Allowed input:

- `import:hrc:preview`
- `import:hrc:dry-run`
- `--help`
- `--dry-run`
- `--command-name <command>`
- `--target-db <copied-db-path>`

The default behavior is still dry-run only. Missing `--dry-run` only produces a warning; it does not enable write behavior.

## Forbidden Flag Policy

The entrypoint rejects:

- `--write`
- `--force`
- `--db-write`
- `--production`
- `--production-db`
- `--raw`
- `--raw-hrc`
- `--raw-hrc-path`
- `--output-json`
- `--report-json`
- `--artifact-json`

Write-like and production flags fail as safety failures. Raw HRC flags fail as privacy/path failures.

The entrypoint always returns:

- `writeAllowed: false`
- `dbWriteAllowed: false`
- `reportFileWriteAllowed: false`

## Path Guard Policy

`--target-db` is treated as a future copied-DB-only preview target. It is validated with the copied DB path guard helper using injected policy strings.

The guard allows only copied DB targets under the approved backup root policy. It rejects production DB targets, repo-local targets, raw HRC path markers, invalid extensions, sidecar files, and private local path exposure according to the guard result.

No path is accessed on disk.

## Output Sanitization Policy

The entrypoint returns `stdoutLines` and `stderrLines`; it does not print them.

Output must not expose:

- raw absolute local paths
- user names or private tokens
- private relay email addresses
- raw HRC source folder paths
- raw HRC zip contents
- external artifact JSON contents

If input contains local/private/raw path text, the returned output uses redacted placeholders.

## Exit Code Policy

- `0`: help or dry-run preview completed without blocking issue.
- `1`: validation blocking issue.
- `2`: safety gate failure or write/production/output flag rejection.
- `3`: privacy/path failure or raw HRC flag rejection.
- `4`: invalid input shape, missing required flag value, unsupported flag, or invalid command.

Manual review candidates remain non-fatal when there are no duplicate, missing-key, privacy, or safety failures.

## Test Coverage

The tests cover:

- `--help` output.
- normal v2.9 fixture dry-run with 19 import-preview allowed rows.
- `import:hrc:preview` and `import:hrc:dry-run`.
- forbidden flag rejection.
- safe copied DB target acceptance.
- production DB target rejection.
- repo-local DB target rejection.
- duplicate existing DB validation exit code.
- safety failure exit code.
- privacy/path failure exit code and redaction.
- invalid input exit code.
- no process exit call.
- no console output call.
- no DB read/write or file write behavior.
