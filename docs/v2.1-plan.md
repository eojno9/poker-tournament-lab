## Action Tree Concept

Action tree는 preflop spot을 단순 position/stack/table size가 아니라 현재 hand가 어떤 action node에 있는지로 분류하는 탐색 context입니다.

예:

- `MTT ICM > 6-max > BTN > 25bb > First-in > RFI 2.2bb`
- `MTT ICM > 6-max > BB > 20bb > Facing Open > vs BTN 2.2bb`
- `MTT ICM > 8-max > SB > 15bb > Push/Fold`
- `MTT ICM > 6-max > CO > 30bb > Facing Limp`

v2.1은 action tree를 solver처럼 새로 계산하지 않습니다. 기존 DB row의 metadata, actionPath, treeConfig, source metadata, strategy actions[]를 읽어 Browser에서 선택 가능한 read-only 분류 정보를 만듭니다.

## Spot Type Classification

### Push/Fold

- all-in/shove/fold 중심 spot입니다.
- `OPEN_SHOVE_ONLY`, `PUSH_FOLD`, `SHOVE_ONLY`, `OPEN_SHOVE`, `SHOVE`, `ALL_IN`, `jam`, `all-in` 같은 actionPath/treeConfig/source token을 신호로 봅니다.
- availableActions에 FOLD / SHOVE / ALL_IN 계열이 있으면 보조 신호로 사용합니다.

### RFI / Open Raise

- first-in 또는 unopened pot에서 open raise/raise-first-in 결정을 하는 spot입니다.
- `RFI`, `OPEN_RAISE`, `FIRST_IN`, `raise first in` 같은 token을 신호로 봅니다.
- open size가 있으면 breadcrumb와 size filter에 표시합니다.

### Limp

- first-in limp 또는 limp option을 가진 spot입니다.
- `LIMP`, `OPEN_LIMP`, `FIRST_IN_LIMP` 같은 token과 strategy action kind를 신호로 봅니다.
- LIMP는 CALL과 합치지 않고 별도 display action으로 취급합니다.

### Facing Open

- hero 앞에 open raise가 있고 hero가 call/raise/fold/all-in 등을 결정하는 spot입니다.
- `FACING_OPEN`, `VS_OPEN`, `CALL_OPEN`, `DEFEND_VS_OPEN`, `vs raise` 같은 token을 신호로 봅니다.

### Facing Limp

- hero 앞에 limp가 있고 hero가 isolate raise/check/call/fold 등을 결정하는 spot입니다.
- `FACING_LIMP`, `VS_LIMP`, `ISO_VS_LIMP`, `isolation` 같은 token을 신호로 봅니다.

### 3bet / vs 3bet

- hero가 3bet을 하거나 3bet을 facing하는 node입니다.
- `THREE_BET`, `3BET`, `VS_3BET`, `FACING_3BET` 같은 token을 신호로 봅니다.
- 3bet을 단순 RAISE로만 취급하지 않고 actionPath/treeConfig/source 신호를 우선합니다.

## LIMP vs CALL Principle

- LIMP는 unopened/first-in pot에서 BB를 콜하는 액션입니다.
- CALL은 이미 bet/raise가 있는 상황에서 따라가는 액션입니다.
- action kind가 LIMP로 제공되면 CALL로 합치지 않습니다.
- action kind가 CALL뿐이고 first-in limp metadata가 있으면 warning과 함께 LIMP 후보로 분류할 수 있습니다.
- source가 불명확하면 임의로 LIMP로 바꾸지 않고 warning을 남깁니다.

## Browser UI Direction

### Spot Type Filter

- Browser Spot Selector에서 DB에 실제 존재하는 spot type만 표시합니다.
- 예: `ALL`, `Push/Fold`, `RFI / Open Raise`, `Limp`, `Facing Open`, `Facing Limp`, `3bet`, `vs 3bet`, `Unknown`
- 없는 spot type을 option으로 만들지 않습니다.

### Action Node Filter

- Browser Spot Selector에서 DB에 실제 존재하는 action node만 표시합니다.
- 예: `ALL`, `Open shove`, `First-in`, `Open raise`, `Open limp`, `Vs open`, `Vs limp`, `3bet`, `Vs 3bet`, `Unknown`
- nearest recommendation이나 가상 node 보간은 하지 않습니다.

### Available Action / Size Filter

- strategy actions[] 또는 legacy adapter 결과에서 실제 존재하는 action/size만 표시합니다.
- action kind filter와 size label filter는 selected solution의 Browser v2 model과 action tree context를 함께 보여줍니다.
- size 없는 RAISE/BET/CALL은 `사이즈 미지정`으로 표시할 수 있습니다.
- 없는 size를 임의 생성하지 않습니다.

### Action Tree Breadcrumb

- 현재 selected solution의 action tree context를 compact breadcrumb로 표시합니다.
- 예: `HRC Smoke DB > 6-max > BTN > 20bb > RFI / Open Raise > Open raise`
- breadcrumbItems가 비어 있으면 `Action tree 정보 제공되지 않음`을 표시합니다.

### Node Candidate Summary / Filter Context

- Spot Selector 주변에 현재 필터 기준 후보 수를 표시합니다.
- 표시 항목:
  - Candidate Solutions
  - Current Node
  - Available Actions
  - Available Sizes
  - Filtered by
- action/size filter context는 현재 action kind filter, size label filter, node available actions/sizes, model action/size options를 표시합니다.
- 후보가 0개면 현재 필터, DB-only, no nearest recommendation 안내를 표시합니다.

### Matrix / Hand Detail Node Context

- Strategy Matrix 제목 주변에 현재 Spot Type / Action Node context를 표시합니다.
- Hand Detail Panel에도 selected hand가 어떤 node의 hand인지 표시합니다.
- 필터 결과가 없으면 현재 action/size filter와 함께 `선택한 필터에 해당하는 action이 없습니다`를 표시합니다.
