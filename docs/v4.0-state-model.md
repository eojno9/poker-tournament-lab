# v4.0 state model

## Purpose

This document defines stable user-facing state categories for v4.0. The goal is
consistent copy, reset behavior, and test expectations across Analyze, Database,
Browser, Trainer, and verification surfaces.

## State Table

| State | User Meaning | Next Action | Data Preservation | Reset Scope | Test Baseline |
| --- | --- | --- | --- | --- | --- |
| idle | No active operation has started. | Choose a tab, enter filters, or start a local action. | Preserve existing local records. | None unless the user resets a specific form. | Initial render and empty recent sections. |
| ready | Inputs are sufficient for a manual action. | Run Analyze, choose a Database row, or start/continue Trainer. | Preserve form and local storage. | Specific form or filter reset only. | Form controls enabled and labeled. |
| loading/processing | A request or local processing step is running. | Wait; avoid duplicate submit when button is disabled. | Preserve current inputs. | No broad reset. | Loading copy and disabled state. |
| success | A manual action finished and has results. | Review result, save preset, continue Trainer, or inspect details. | Save only intended local records. | Result-specific reset. | Result sections visible. |
| empty | Query/filter has no rows or local list has no entries. | Adjust filters, clear filters, or add local records. | Preserve filters unless user clears them. | Filter/list scope only. | Empty state copy is Korean-first. |
| invalid | Input is malformed or incomplete. | Fix highlighted fields. | Preserve user input. | Field/form scope only. | Validation messages are visible. |
| not solved | Exact result is unavailable and fallback cannot run or is not applicable. | Adjust study spot or review missing requirements. | Preserve request and recent state. | Result scope only. | `NOT_SOLVED` remains explicit. |
| unavailable | Optional local source or artifact is unavailable. | Check local setup or continue with another section. | Preserve unrelated data. | Source-specific scope. | Unavailable copy avoids hidden write paths. |
| recoverable error | A request, parse, or storage operation failed but the app can continue. | Retry, adjust input, or continue without saved local data. | Preserve current in-memory session when possible. | Failed operation scope only. | Error copy avoids raw payload/path disclosure. |
| completed | A session or workflow unit is finished. | Review summary, start a new session, or reset the specific flow. | Preserve historical local records unless explicit clear is used. | Session scope only. | Summary and next action visible. |

## Cross-Screen Rules

- A reset action must name its scope or be visually located in that scope.
- Handoff context should be visible until the user clears it.
- No state should auto-run Analyze after Database handoff.
- Empty/error/not-solved states should not recommend nearest matches as exact
  results.
- Storage failure should not crash Trainer or Analyze.
- User-facing state copy should be Korean-first, with technical constants kept
  only where exact terminology matters.

## v4.0 Must Checks

- Repair any mojibake state copy before release.
- Confirm each major screen has at least idle/ready, empty, error, and success
  or completed coverage where applicable.
- Confirm state copy does not expose local paths, raw payloads, secrets, or data
  source details.

## Implementation Application Result

The v4.0 implementation bundle applied the state model through focused cleanup
rather than a new global state machine.

Confirmed results:

- SECURITY.md recoverable reporting guidance is readable and public-safe.
- Trainer missing tree configuration uses the Korean fallback `제공되지 않음`.
- Database-to-Analyze handoff continues to preserve editable context and does
  not auto-run Analyze.
- Handoff reset remains scoped to the handoff notice and does not clear presets,
  recent analyses, Trainer history, or unrelated local records.
- Existing smoke coverage continues to exercise empty, not-solved, error,
  handoff, Trainer, and safety-copy states.
- No raw payload, local path, secret, or data-origin detail is intentionally
  exposed through the touched user-facing state copy.
