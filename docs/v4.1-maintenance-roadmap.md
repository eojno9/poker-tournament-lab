## Node SQLite Warning

The server and compatibility tests import `node:sqlite`. Node.js may emit an
experimental warning even when behavior is correct.

Classification: `maintenance only`, non-blocking.

Recommended handling:

- keep the warning visible
- avoid replacement dependencies solely to remove the warning
- re-check upstream Node status during a future runtime maintenance review
- treat behavior or compatibility failures, not the warning text alone, as a
  blocker

## Public Repository Upkeep

For each public release:

- use a fresh sanitized upload folder
- copy only the approved public-safe snapshot subset
- verify README and RELEASE_HISTORY current status before the snapshot commit
- run typecheck/test/build/smoke in the public folder
- run public-safe and forbidden-artifact scans
- confirm normal fast-forward push is possible
- confirm the candidate remote tag is absent
- create and push only the named tag
- verify Actions and rendered README after push

Post-push documentation should avoid exact external facts that force a status
commit unless that audit value is worth the extra main-only hotfix.

## Safety Boundary

No roadmap item adds live assistance, client integration, bundled data,
server-side user persistence, automatic push, force push, or tag overwrite.
