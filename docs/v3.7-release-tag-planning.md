## Release Summary

v3.7 completes the remaining practical Trainer workflow work before a later v4.0
readiness review. It keeps Trainer state local to the browser and does not
expand the product into live-play assistance or server persistence.

Included:

- local session status for start, progress, completion, and reset
- session completion summary
- local filter save/load/reset UX
- versioned filter settings localStorage key: `ptl.trainer.filters.v1`
- safe fallback for corrupt, partial, unknown-version, and write-failure filter payloads
- mistake review status filters for all, unresolved, resolved, and hidden records
- filtered mistake empty states
- Korean-first UI copy for touched Trainer surfaces
- unit and smoke coverage for the new workflows

Excluded:

- server persistence
- account or cloud storage
- API route changes
- DB writes
- raw data handling
- generated report workflows
- new dependencies
- package script changes
- live/RTA/OCR/screen capture/overlay/hotkey/watcher/client integration
